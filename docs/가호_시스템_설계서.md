# 가호(加護) 시스템 설계서

> 버전: 1.1 | 작성일: 2026-04-19 | 구현 완료: 2026-04-19
> 대상: GahoSystem.js (신규 플러그인) + RMMZStudio.html UI 확장

---

## 1. 개요

### 1.1 목적
사주팔자 기반의 세계관 시스템 "가호록"을 RMMZ 플러그인으로 구현.
각 액터에 **원국(사주 4주 8글자)**을 부여하고, 이로부터 성격·기질·특수 태그를 자동 파생하여
게임 내 캐릭터 정체성과 전투 스탯 보정의 기반 데이터로 활용한다.

### 1.2 세계관 변환 규칙
- 10천간 → **10성좌** (캐릭터의 "추구점")
- 12지지 → **12주신** (캐릭터의 "타고난 기질")
- 4주(년·월·일·시) → 각각 가문/가족/본인/하급자 영향
- 오행 상생상극 → 기질 축 계산의 기반
- 음양 매칭: 양간↔양지, 음간↔음지 (기둥 내 필수)

### 1.3 핵심 기능
1. **원국 저장**: 액터별 4주 원국 + 종족 + 성별을 플러그인 관리 데이터로 저장
2. **파생 계산**: 원국으로부터 15축 기질 파라미터, 신살 태그, 아키타입 자동 도출
3. **랜덤 생성**: 🎲 버튼으로 종족별 생년월일시 랜덤 → 원국 자동 계산
4. **설계 생성**: 🔧 모달에서 원하는 성격 축 슬라이더 → 원국 역산 생성
5. **SRPG API**: SrpgUnit이 원국 데이터를 읽어 전투 스탯 보정에 활용 가능

## 2. 데이터 모델

### 2.1 저장 구조 — 액터 note 태그

```
<gahoPillars:甲子|丙寅|戊辰|壬午>    // 4주 (시|일|월|년 순, 간지 한자 쌍)
<gahoRace:human>                     // 종족 ID
<gahoGender:m>                       // 성별 (m/f)
<gahoBirth:1180,3,15,7>              // 생년월일시 (오르비아력 기준, 선택)
```

- **gahoPillars**: 핵심 데이터. 4주 × 2글자(간+지) = 8글자. 한자 TID로 저장하여 정규화.
  - 순서: `시주간지|일주간지|월주간지|년주간지`
  - 예: `甲子|丙寅|戊辰|壬午`
- **gahoRace**: 종족 ID (human/elf/orc/ogre/dwarf/darkelf)
- **gahoGender**: 성별 (m/f) — H기질 타이틀 분기 및 신살(고신/과숙) 판정
- **gahoBirth**: 오르비아력 생년,월,일,시 — 랜덤/달력 기반 생성 시 기록 (설계 모드는 생략 가능)

### 2.2 런타임 데이터 (파생 — 저장하지 않음)

| 카테고리 | 내용 | 계산 원천 |
|---------|------|----------|
| 4주 상세 | 성좌명, 주신명, 오행, 음양, 이모지 | TID → 매핑 테이블 |
| 오행 비율 | {wood, fire, earth, metal, water} 각 % | 8슬롯 오행 합산 |
| 십성 5축 | 주체성/사교성/현실감각/자기절제/계획성 | 일간 기준 십성 카운트 |
| 오행 5축 | 공감능력/몰입능력/정서안정/판단력/학습능력 | 오행 비율 기반 |
| H기질 5축 | 정조관념/도덕성/쾌락내성/지배성향/가학성향 | 십성×오행 복합 |
| 신살 태그 | 위기의 은인, 천재 괴짜 등 25종+ | 간지 조합 룩업 |
| 아키타입 | 야심가, 충성의 칼, 자유영혼 등 18종 | 십성 카운트 조합 |
| 시놉시스 | 자연어 성격 요약 텍스트 | 전체 파생 데이터 종합 |

### 2.3 종족 시스템

| 종족 ID | 이름 | 나이비율 | 년지 규칙 | 나이 범위 |
|---------|------|---------|----------|----------|
| human | 인간 | 1.0 | 일반 60갑자 순환 | 16-45 |
| elf | 엘프 | 0.1 | 木 고정 (양=인, 음=묘) | 160-400 |
| orc | 오크 | 2.0 | 火 고정 (양=오, 음=사) | 8-30 |
| ogre | 오거 | 0.4 | 土 랜덤 (양=진/술, 음=축/미) | 40-150 |
| dwarf | 드워프 | 0.67 | 金 고정 (양=신, 음=유) | 24-65 |
| darkelf | 다크엘프 | 0.1 | 水 고정 (양=자, 음=해) | 160-400 |

- `나이비율`: 외양나이 = 실제나이 × 비율
- `년지 규칙`: 종족별로 년주 지지가 오행 고정 (천간 음양에 따라 양지/음지 결정)

### 3.2 사주 계산 (생년월일시 → 4주)

```
calcSajuFromDate(year, month, day, hour, race) → pillars[4]

년주: (year-1) % 60 → 간 = GANS[idx%10], 지 = 종족별 규칙 또는 JIS[idx%12]
일주: absDay(year,month,day) → (absDay-1) % 60 → 간 = [idx%10], 지 = [idx%12]
월주: 년상기월법 (년간 기준 월간 계산)
시주: 일상기시법 (일간 기준 시간 계산)
```

### 3.3 십성 5축 (행동 패턴 · 관계 구조)

일간 오행 기준, 나머지 7슬롯의 오행과의 관계를 십성으로 분류:

| 축 ID | 축명 | 십성 | 관계 | over | under |
|-------|------|------|------|------|-------|
| bigup | 주체성 | 비겁 | 나와 같은 오행 | 고집 | 순종 |
| express | 사교성 | 식상 | 내가 생하는 오행 | 외향 | 내향 |
| percept | 현실감각 | 재성 | 내가 극하는 오행 | 현실 | 망상 |
| discip | 자기절제 | 관성 | 나를 극하는 오행 | 경직 | 방탕 |
| reflect | 계획성 | 인성 | 나를 생하는 오행 | 신중 | 즉흥 |

카운트: 7슬롯 중 해당 십성 수 (0~7). balanced = total / 5.
deviation = (count - balanced) / balanced → 바 위치 결정.

### 3.4 오행 5축 (본질 에너지)

오행 비율(각 0~50%) 기반. balanced = 20%.

| 축 ID | 축명 | 오행 | over | under |
|-------|------|------|------|-------|
| wood | 공감능력 | 木 | 자비 | 냉담 |
| fire | 몰입능력 | 火 | 열정 | 냉소 |
| earth | 정서안정 | 土 | 안정 | 불안 |
| metal | 판단력 | 金 | 결단 | 우유부단 |
| water | 학습능력 | 水 | 지혜 | 둔감 |

### 3.5 H기질 5축 (십성×오행 파생)

| 축 ID | 축명 | 계산 공식 요약 |
|-------|------|--------------|
| chastity | 정조관념 | 3 + 관성↑×1.2 + 金↑×0.8 + 재성↓×0.7 |
| morality | 도덕성 | 3 + 木↑×1.0 + 관성↑×0.8 + 재성↓×0.7 |
| resist | 쾌락내성 | 3 + 인성↑×1.0 + 관성↑×0.8 + 水↓×0.5 + 火↓×0.5 |
| submit | 지배성향 | 3 + 비겁↑×1.5 + 土↓×1.0 |
| sadism | 가학성향 | 3 + 비겁↑×0.8 + 재성↑×0.6 + 인성↓×0.5 + 金↑×0.6 + 火↑×0.4 |

각 축 결과: 0~6 정수 (clamp). over=6, under=0, balanced=3.

### 3.6 신살 계산 (25종+)

일간·일지·년지·월지·전체지지 조합에 따른 룩업 테이블.
HTML의 `calcSinsal()` 함수를 그대로 이식.

| 분류 | 신살 예시 | 판정 기준 |
|------|----------|----------|
| 일간→지지 룩업 | 위기의 은인, 책벌레, 치명적 매력 등 | 천을귀인, 문창귀인 등 테이블 |
| 일주 조합 | 천재 괴짜, 빛나는 외모 | 괴강살, 육수 일주 등 |
| 12신살 | 인싸 체질, 파견 전문 | 삼합 기반 12신살 배치표 |
| 복합 조건 | 질긴 악연, 제6감 | 원진살, 귀문관살 등 |
| 성별 분기 | 천생 홀아비/과부 | 고신/과숙 방합 |

### 3.7 아키타입 매칭 (18종)

십성 5축 카운트 조합 패턴 매칭:
```
야심가:      비겁≥3, 식상≥3, 재성≥3
충성의 칼:    비겁=0, 관성≥3, 재성=0
자유영혼:    관성=0, 인성=0
고독한 늑대:  비겁≥3, 식상=0
...
```

## 4. 설계 모달 (역산 생성)

### 4.1 3가지 설계 모드

| 모드 | 설계 대상 | 슬라이더 | 역산 로직 |
|------|----------|---------|----------|
| 십성기질 | 5축 (주체성~계획성) | -3~+3 각 축 | 선호도 → 오행 가중치 → 7슬롯 오행 배분 → 랜덤 간지 선택 |
| 오행기질 | 5축 (공감~학습) | -3~+3 각 축 | 직접 오행 가중치 → 7슬롯 배분 |
| H기질 | 5축 (정조~가학) | -3~+3 각 축 | H→십성/오행 역추적 → 가중치 → 배분 |

### 4.2 역산 알고리즘

```
1. 사용자가 일간 오행 + 성별 + 종족 선택
2. 5축 슬라이더로 선호도 입력 (-3 ~ +3)
3. 선호도 → prefToWeight() → 오행별 가중치 맵 생성
4. 가중치 정규화 → 7슬롯(일간 제외) 오행 배분
5. 배분된 오행에서 랜덤으로 간(3개) + 지(4개) 선택
6. 음양 규칙 검증
7. 4주 조립: [시간|일간+일지|월간|년간] + [시지|일지|월지|년지]
8. note 태그로 저장
```

### 4.3 프리셋

각 모드별 6개 프리셋 (HTML에서 이식):
- 십성: 분석가형, 외교관형, 전사형, 지휘관형, 방랑자형, 균형형
- 오행: 화염술사, 대지의 수호자, 칼날의 심판자, 물의 현자, 숲의 치유사, 균형형
- H기질: 청순가련, 요부형, 철벽녀, 순종미인, 여왕님, 균형형

## 5. 플러그인 구조

### 5.1 GahoSystem.js

```javascript
// 전역 객체
window.GahoSystem = {
    // ─── 매핑 테이블 ───
    GANS: [...],           // 10천간 → 성좌
    JIS: [...],            // 12지지 → 주신
    RACES: {...},          // 6종족 정보
    PARAM_AXES: [...],     // 십성 5축 정의
    ELEM_AXES: [...],      // 오행 5축 정의
    H_AXES: [...],         // H기질 5축 정의
    ARCHETYPES: [...],     // 18종 아키타입
    SINSAL_TABLES: {...},  // 신살 룩업 테이블 전체

    // ─── 사주 계산 ───
    calcSajuFromDate(year, month, day, hour, race),
    parsePillars(pillarStr),   // "甲子|丙寅|..." → [{g, j}, ...]
    encodePillars(pillars),    // [{g, j}, ...] → "甲子|丙寅|..."

    // ─── 파생 계산 ───
    calcElements(pillars),     // → {wood:%, fire:%, ...}
    calcSipsung(pillars),      // → {bigup:n, express:n, ...}
    calcElemAxes(elements),    // → [{id, level, deviation}, ...]
    calcHParams(sipCounts, total, elements), // → {chastity:0-6, ...}
    calcSinsal(pillars, gender),   // → {name: level, ...}
    matchArchetypes(sipCounts),    // → [{name, icon}, ...]
    generateSynopsis(actorData),   // → 시놉시스 텍스트

    // ─── 생성 ───
    randomGenerate(race, gender),         // 랜덤 생성 → pillarStr
    designGenerate(dayElem, gender, mode, prefs), // 설계 역산 → pillarStr

    // ─── 액터 API ───
    getActorData(actorId),     // note 태그에서 원국 읽기 → 전체 파생 포함
    getActorPillars(actorId),  // 4주 raw 데이터만
    getActorElements(actorId), // 오행 비율
    getActorParams(actorId),   // 15축 파라미터
    getActorSinsal(actorId),   // 신살 태그
    getStatModifiers(actorId), // 전투 스탯 보정값 (SRPG 연동용)
};
```

### 5.2 SRPG 연동 API

```javascript
// SrpgUnit에서 호출 가능
GahoSystem.getStatModifiers(actorId)
// → { atk: +5, def: -2, spd: +3, ... }  (추후 구체화)

// 전투 시 오행 상성 보정
GahoSystem.getElementAffinity(attackerActorId, defenderActorId)
// → { bonus: 1.2, relation: 'gen' }  (추후 구체화)
```

### 6.2 설계 모달

HTML의 설계 모달을 RMMZStudio 스타일로 이식:
- 일간 오행 5버튼 선택
- 성별 2버튼
- 종족 드롭다운
- 설계 기준 3탭 (십성기질 / 오행기질 / H기질)
- 5축 슬라이더 (-3 ~ +3) + 실시간 타이틀 피드백
- 프리셋 버튼 행
- [생성] 버튼 → 원국 역산 → note 태그 저장 → UI 갱신

### 6.3 노트 태그 동기화

기존 `syncSrpgGridNote()` 패턴과 동일:
- UI 변경 → `syncGahoNote()` → 기존 가호 태그 제거 → 새 태그 생성 → item.note 갱신
- 폼 렌더링 시 note에서 태그 파싱 → UI 필드 채우기
