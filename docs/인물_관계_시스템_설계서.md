# 인물 관계 시스템 설계서 (Relationship System v2)

> **작성일**: 2026-04-21  
> **상태**: 설계 단계 (구현 전)  
> **대상 플러그인**: GridInventory.js (기존 관계 코드 교체) + 신규 RelationSystem.js

---

## 1. 개요

기존 2축(호감도 favor / 신뢰도 trust) 대칭형 시스템을 폐기하고, 3축 비대칭 관계 시스템으로 재설계한다.

| 축 | 이름 | 성격 | 핵심 역할 |
|---|---|---|---|
| **Impression** | 인상 점수 | 비대칭 상대값 (A→B ≠ B→A) | 상대에 대한 현재 감정적 거리 |
| **Mood** | 기분 상태 | 개인 속성 (절대값) | 비전투 상호작용 순응 확률에 영향 |
| **Bond Tag** | 관계 태그 | 방향성 있는 라벨 (A→B) | 관계의 질적 정의 + 인상 점수 모디파이어 |

**설계 원칙**:
- 인상은 점수일 뿐, 관계의 정의가 아니다. 관계의 정의는 Bond Tag가 담당한다.
- 기분은 주변 인물의 인상 점수에서 파생되며, 전투가 아닌 일상 상호작용에만 영향을 미친다.
- Bond Tag 전환은 자동이 아니라 이벤트 스크립트로만 발생한다. "인간관계는 점수보다 특정 계기가 크다."

## 2. Impression (인상 점수)

### 2.1 데이터 구조

```javascript
// Game_Actor._impressions = { targetActorId: number }
// 예시: { 2: 35, 5: -20, 8: 0 }
```

- **범위**: -100 ~ +100 (정수)
- **기본값**: 0 (초면 시)
- **비대칭**: Actor A → B 와 Actor B → A 는 완전히 독립적인 값

### 2.2 변동 요인

| 트리거 | 변동량 (예시) | 방향 |
|---|---|---|
| 전투 중 회복/버프 제공 | +3 ~ +8 | 수혜자 → 시전자 |
| 전투 중 같은 적 공격 (협공) | +1 ~ +3 | 양방향 |
| 아이템 증여 | +5 ~ +15 (아이템 가치 비례) | 수령자 → 증여자 |
| 대화 선택지 (호의적) | +2 ~ +10 | 대상 → 플레이어 |
| 대화 선택지 (적대적) | -5 ~ -15 | 대상 → 플레이어 |
| 같은 파티에서 전투 완료 | +1 | 양방향 (파티원 전체) |
| 투척으로 피해 (아군오사) | -10 ~ -20 | 피해자 → 시전자 |

### 2.3 Bond Tag 모디파이어

Bond Tag가 인상 점수 변동에 승수를 부여한다:

```
최종 변동량 = 기본 변동량 × bondModifier
```

| Bond Tag | 긍정 변동 승수 | 부정 변동 승수 | 설명 |
|---|---|---|---|
| stranger (초면) | ×1.0 | ×1.0 | 기본값 |
| acquaintance (지인) | ×1.1 | ×0.9 | 약간의 호의적 편향 |
| friend (친구) | ×1.3 | ×0.7 | 긍정은 크게, 부정은 둔감 |
| rival (라이벌) | ×0.8 | ×1.3 | 부정에 민감, 긍정에 둔감 |
| sworn (맹우) | ×1.5 | ×0.5 | 강한 신뢰 보정 |
| hostile (적대) | ×0.5 | ×1.5 | 강한 불신 보정 |

### 2.4 자연 감쇠 (Decay)

극단적인 인상값은 시간이 지나면 중앙(0)을 향해 서서히 감쇠한다.

```
매 게임 내 1일 경과 시:
  if |impression| > 30:
    impression -= sign(impression) × 1
```

- 절대값 30 이하는 감쇠하지 않는다 (일상적 범위).
- Bond Tag가 sworn/friend이면 양수 방향 감쇠를 차단한다 (관계가 깊으면 호감이 유지).
- Bond Tag가 hostile이면 음수 방향 감쇠를 차단한다.

## 3. Mood (기분 상태)

### 3.1 데이터 구조

```javascript
// Game_Actor._mood = number
// 범위: -100 ~ +100, 기본값: 0 (보통)
```

- 개인 속성 (방향 없음)
- 전투 외 상호작용에만 영향

### 3.2 기분 계산

기분은 **주변 인물의 인상 점수** 로부터 매 장면(맵 전환, 전투 종료 등) 갱신된다.

```javascript
function recalcMood(actor) {
    const party = $gameParty.members();
    let sum = 0;
    let count = 0;
    for (const member of party) {
        if (member === actor) continue;
        const imp = actor.getImpression(member.actorId());
        sum += imp;
        count++;
    }
    // 파티원 평균 인상이 기분의 기저
    const base = count > 0 ? Math.round(sum / count) : 0;
    
    // 기존 기분에서 기저를 향해 부드럽게 이동 (관성)
    const current = actor.mood();
    const next = current + Math.round((base - current) * 0.3);
    
    actor.setMood(Math.max(-100, Math.min(100, next)));
}
```

**핵심**: 기분은 즉각 변하지 않고 관성이 있다. 갑자기 좋아하는 인물이 파티에 들어와도 기분이 서서히 좋아진다.

### 3.3 추가 기분 요인

| 트리거 | 기분 변동 | 비고 |
|---|---|---|
| 전투 승리 | +5 | 파티원 전체 |
| 전투 패배 (퇴각) | -10 | 파티원 전체 |
| 아이템 수령 | +3 | 해당 인물만 |
| HP 30% 이하 상태 유지 | -5/장면 | 부상 스트레스 |
| 호감 높은 인물(imp > 50) 전사/퇴장 | -20 | 해당 인물만 |

### 3.4 기분 → 순응 확률

비전투 상호작용(아이템 교환 요청, 위치 이동 지시, 대화 선택지 반응)에서:

```
기본 순응률 = 70%

기분 보정:
  mood > 0:  순응률 += mood × 0.3  (최대 +30%)
  mood < 0:  순응률 += mood × 0.5  (최대 -50%)

Bond Tag 보정:
  friend/sworn → +10%
  rival/hostile → -15%

최종 순응률 = clamp(기본 + 기분보정 + 태그보정, 5%, 99%)
```

**예시**:
- 기분 +60, 친구 관계: 70 + 18 + 10 = **98%** (거의 확정 순응)
- 기분 -40, 라이벌 관계: 70 - 20 - 15 = **35%** (불만족, 반발 가능)
- 기분 -80, 적대 관계: 70 - 40 - 15 = **15%** (극단적 비협조)

**불응(순응 실패) 시 행동**:
- 아이템 교환 → 거절 메시지 + 잠깐 불만 표정
- 이동 지시 → 무시하거나 다른 방향으로 이동
- 대화 → 비꼬는 응답 변형

## 4. Bond Tag (관계 태그)

### 4.1 태그 분류

Bond Tag는 방향성이 있다 (A→B). A가 B를 "친구"로 여겨도 B는 A를 "라이벌"로 볼 수 있다.

#### 기본 진행 트랙

```
stranger → acquaintance → friend → sworn
  (초면)     (지인)        (친구)    (맹우)
```

#### 특수 트랙

```
stranger → acquaintance → rival → hostile
                           (라이벌)  (적대)

acquaintance → mentor/pupil (사제)
friend → lover (연인) [해당하는 경우]
any → broken (결별) [되돌릴 수 없는 배신 등]
```

### 4.2 태그 정의

| Tag ID | 한글명 | 인상 변동 승수 (긍정/부정) | 기분 순응 보정 | 비고 |
|---|---|---|---|---|
| `stranger` | 초면 | 1.0 / 1.0 | ±0% | 기본값 |
| `acquaintance` | 지인 | 1.1 / 0.9 | +3% | 얼굴은 아는 사이 |
| `friend` | 친구 | 1.3 / 0.7 | +10% | 신뢰 관계 |
| `rival` | 라이벌 | 0.8 / 1.3 | -15% | 경쟁적 긴장 |
| `sworn` | 맹우 | 1.5 / 0.5 | +10% | 생사를 같이한 전우 |
| `hostile` | 적대 | 0.5 / 1.5 | -15% | 적의 관계 |
| `mentor` | 스승 | 1.2 / 0.8 | +8% | 가르침 관계 (A→B: A의 스승이 B) |
| `pupil` | 제자 | 1.2 / 0.8 | +5% | mentor의 역방향 |
| `lover` | 연인 | 1.5 / 1.5 | +15% | 긍정도 부정도 크다 |
| `broken` | 결별 | 0.3 / 2.0 | -20% | 돌이킬 수 없는 파탄 |

### 4.3 태그 전환 API

Bond Tag는 **이벤트 스크립트에서만** 변경 가능하다. 자동 전환은 없다.

```javascript
// 플러그인 커맨드
SetBondTag actorId targetId tag
// 예: SetBondTag 1 3 friend  → Actor 1이 Actor 3을 "friend"로 설정

// 스크립트 콜
$gameActors.actor(1).setBondTag(3, 'friend');

// 이벤트에서의 활용 예
// "위기에서 구출" 이벤트 이후:
// ◆스크립트: $gameActors.actor(1).setBondTag(3, 'sworn');
// ◆스크립트: $gameActors.actor(3).setBondTag(1, 'sworn');
```

### 4.4 태그 히스토리 (관계의 역사)

Bond Tag 변경 시 이전 태그와 변경 사유를 기록한다. 이것이 "관계의 역사"이다.

```javascript
// Game_Actor._bondHistory = {
//   targetActorId: [
//     { tag: 'stranger', since: 0 },
//     { tag: 'acquaintance', since: 15, reason: '첫 대화' },
//     { tag: 'friend', since: 42, reason: '함께 위기를 넘김' },
//   ]
// }
```

- `since`: 게임 내 경과 일수 (또는 턴 수)
- `reason`: 이벤트에서 전달하는 텍스트 (선택사항)
- UI에서 "관계 이력"으로 열람 가능 (향후 구현)

### 5.1 기존 형식 (폐기 예정)

```
<srpgRelation:targetActorId,favor,trust>
```

### 5.2 신규 형식

```
<srpgRelation:targetActorId,impression,bondTag>
```

**예시**:
```
<srpgRelation:2,30,acquaintance>
```

- `impression`: 초기 인상 점수 (-100 ~ 100)

- Mood는 노트태그에 저장하지 않는다 (런타임 계산값)

| 기존 | 신규 |

| favor 입력 (-100~100) | impression 입력 (-100~100) |
| trust 입력 (-100~100) | bondTag 드롭다운 (태그 목록) |

## 6. 런타임 API 설계

### 6.1 Game_Actor 확장 (GridInventory.js 내 기존 코드 교체)

```javascript
// ─── Impression ───
Game_Actor.prototype.initRelations = function() {
    this._impressions = {};      // { actorId: number }
    this._bondTags = {};         // { actorId: string }
    this._bondHistory = {};      // { actorId: [{tag, since, reason}] }
    this._mood = 0;
};

Game_Actor.prototype.getImpression = function(targetId) {
    return this._impressions[targetId] || 0;
};

Game_Actor.prototype.setImpression = function(targetId, value) {
    this._impressions[targetId] = Math.max(-100, Math.min(100, Math.round(value)));
};

// Bond Tag 모디파이어 적용
Game_Actor.prototype.addImpression = function(targetId, amount) {
    const tag = this.getBondTag(targetId);
    const mod = BOND_MODIFIERS[tag] || { pos: 1.0, neg: 1.0 };
    const multiplier = amount >= 0 ? mod.pos : mod.neg;
    const adjusted = Math.round(amount * multiplier);
    this.setImpression(targetId, this.getImpression(targetId) + adjusted);
};

// ─── Bond Tag ───
Game_Actor.prototype.getBondTag = function(targetId) {
    return this._bondTags[targetId] || 'stranger';
};

Game_Actor.prototype.setBondTag = function(targetId, tag, reason) {
    const oldTag = this.getBondTag(targetId);
    if (oldTag === tag) return;
    this._bondTags[targetId] = tag;
    // 히스토리 기록
    if (!this._bondHistory[targetId]) {
        this._bondHistory[targetId] = [{ tag: 'stranger', since: 0 }];
    }
    this._bondHistory[targetId].push({
        tag: tag,
        since: $gameVariables.value(/* 경과일수 변수 ID */),
        reason: reason || ''
    });
};

Game_Actor.prototype.getBondHistory = function(targetId) {
    return this._bondHistory[targetId] || [];
};

// ─── Mood ───
Game_Actor.prototype.mood = function() {
    return this._mood || 0;
};

Game_Actor.prototype.setMood = function(value) {
    this._mood = Math.max(-100, Math.min(100, value));
};

Game_Actor.prototype.addMood = function(amount) {
    this.setMood(this.mood() + amount);
};

// ─── 순응 확률 ───
Game_Actor.prototype.complianceRate = function(targetId) {
    const BASE = 70;
    const mood = this.mood();
    const moodBonus = mood > 0 ? mood * 0.3 : mood * 0.5;
    const tag = targetId ? this.getBondTag(targetId) : 'stranger';
    const tagBonus = COMPLIANCE_BONUS[tag] || 0;
    return Math.max(5, Math.min(99, Math.round(BASE + moodBonus + tagBonus)));
};
```

### 6.2 플러그인 커맨드 (기존 교체)

| 커맨드 | 인자 | 설명 |
|---|---|---|
| `SetImpression` | actorId, targetId, value | 인상 점수 직접 설정 (편방향) |
| `AddImpression` | actorId, targetId, amount | 인상 점수 변동 (모디파이어 적용, 편방향) |
| `AddImpressionBi` | actorId, targetId, amountAB, amountBA | 양방향 인상 변동 (각각 다른 값 가능) |
| `SetBondTag` | actorId, targetId, tag, reason | Bond Tag 변경 (편방향, 히스토리 기록) |
| `SetMood` | actorId, value | 기분 직접 설정 |
| `AddMood` | actorId, amount | 기분 변동 |
| `RecalcMood` | actorId | 파티 인상 기반 기분 재계산 |

**비대칭 핵심**: 기존의 양방향 강제 적용 폐기. 양방향이 필요하면 `AddImpressionBi`를 사용하되, 각 방향의 변동량을 개별 지정할 수 있다.

### 6.3 상수 테이블

```javascript
const BOND_MODIFIERS = {
    stranger:     { pos: 1.0, neg: 1.0 },
    acquaintance: { pos: 1.1, neg: 0.9 },
    friend:       { pos: 1.3, neg: 0.7 },
    rival:        { pos: 0.8, neg: 1.3 },
    sworn:        { pos: 1.5, neg: 0.5 },
    hostile:      { pos: 0.5, neg: 1.5 },
    mentor:       { pos: 1.2, neg: 0.8 },
    pupil:        { pos: 1.2, neg: 0.8 },
    lover:        { pos: 1.5, neg: 1.5 },
    broken:       { pos: 0.3, neg: 2.0 },
};

const COMPLIANCE_BONUS = {
    stranger: 0, acquaintance: 3, friend: 10, rival: -15,
    sworn: 10, hostile: -15, mentor: 8, pupil: 5,
    lover: 15, broken: -20,
};
```

## 7. 시스템 상호작용 예시

### 예시 1: "첫 만남에서 맹우까지"

```
[1일차] Actor A, B 첫 조우
  A→B impression: 0, bondTag: stranger
  B→A impression: 0, bondTag: stranger

[3일차] 같은 파티로 전투 3회 → A→B +3, B→A +3
  A→B impression: 3

[5일차] 이벤트: A가 B에게 아이템 증여(가치 높음) → B→A +12
  B→A impression: 15
  * A→B는 변동 없음 (비대칭!)

[5일차] 이벤트 스크립트: 대화 후 지인으로 전환
  SetBondTag A→B acquaintance ("첫 대화")
  SetBondTag B→A acquaintance ("첫 대화")

[12일차] 전투에서 B가 A 회복 → A→B +6 (×1.1 지인 보정 = +6.6 → +7)
  A→B impression: 10

[20일차] 이벤트: 함께 위기 돌파 → 친구로 전환
  SetBondTag A→B friend ("생사의 고비")
  SetBondTag B→A friend ("생사의 고비")

[35일차] 이벤트: 결정적 전투에서 서로를 구함 → 맹우
  SetBondTag A→B sworn ("등을 맡기다")
  A→B impression: 65 (그간 누적)
```

### 예시 2: "기분이 전략에 미치는 영향"

```
현재 파티: [주인공, A, B, C]

주인공→A impression: +60 (친구)
주인공→B impression: -30 (라이벌)
주인공→C impression: +10 (지인)

주인공 기분 계산:
  평균 = (60 + (-30) + 10) / 3 = 13.3
  기분: 이전 0 → 0 + (13-0)×0.3 = +4 (서서히 상승)

--- 수 턴 경과, 기분 안정 후 ---
  기분: +13

주인공이 B에게 아이템 교환 요청:
  기본 순응률: 70%
  기분 보정: +13 × 0.3 = +3.9
  B→주인공 태그: rival → -15%
  최종: 70 + 4 - 15 = 59% → 실패 가능성 높음

주인공이 A에게 같은 요청:
  기분 보정: +4
  A→주인공 태그: friend → +10%
  최종: 70 + 4 + 10 = 84% → 높은 성공률
```

## 8. 기존 시스템과의 마이그레이션

### 8.1 데이터 변환

| 기존 | 신규 | 변환 규칙 |
|---|---|---|
| `favor` | `impression` | 그대로 매핑 |
| `trust` | (폐기) | Bond Tag 초기값 결정에 참고 |
| (없음) | `bondTag` | trust ≥ 30 → acquaintance, trust ≥ 60 → friend, 기본 stranger |
| (없음) | `mood` | 런타임 계산, 저장 불필요 |

### 8.2 호환 레이어

마이그레이션 기간 동안 기존 API를 래핑:

```javascript
// 하위 호환 (경고 로그 출력 후 신규 API 호출)
Game_Actor.prototype.addFavor = function(targetId, amount) {
    console.warn('addFavor deprecated → use addImpression');
    this.addImpression(targetId, amount);
};
Game_Actor.prototype.addTrust = function(targetId, amount) {
    console.warn('addTrust deprecated → Bond Tags replace trust');
};
```

### 8.3 세이브 데이터

`_relationships` 객체를 `_impressions` + `_bondTags` + `_bondHistory` + `_mood`로 분리. 기존 세이브 로드 시 마이그레이션 함수 실행:

```javascript
Game_Actor.prototype.migrateRelations = function() {
    if (!this._relationships) return;
    for (const [id, rel] of Object.entries(this._relationships)) {
        this._impressions[id] = rel.favor || 0;
        // trust → bondTag 추정
        const trust = rel.trust || 0;
        if (trust >= 60) this._bondTags[id] = 'friend';
        else if (trust >= 30) this._bondTags[id] = 'acquaintance';
    }
    delete this._relationships;
};
```
