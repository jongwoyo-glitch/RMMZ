# 멀티타일 유닛 & 오브젝트 시스템 설계서

> 버전: 2.0 | 작성일: 2026-04-19 | 구현 완료: 2026-04-19
> 대상: SRPG_Core (SRPG_Data.js, SRPG_SM.js, SRPG_UI.js)

---

## 1. 개요

### 1.1 현재 상태
- 모든 유닛(액터/적/오브젝트)이 1×1 타일만 점유
- `SrpgUnit.x`, `.y`는 단일 좌표 (RMMZ 이벤트 위치)
- `isOccupied(x,y)`는 해당 좌표에 유닛이 있는지만 확인
- 오브젝트(`<srpgObject:true>`)는 이벤트 노트태그로 직접 정의 → 재사용/관리 어려움

### 1.2 목표
1. **멀티타일 점유**: 유닛이 W×H 크기의 타일 영역을 점유
   - 액터/적: 1×1, 2×2, 3×3 (정사각형)
   - 오브젝트: 1×1, 1×2, 1×3, 2×2, 2×3, 1×4, 2×4 (직사각형 포함)
2. **오브젝트 = 액터 DB**: 바리케이드 등을 액터 DB에 등록, 스킬로 소환
3. **스프라이트**: 멀티타일 영역에 이미지가 정확히 맞춰 렌더링
4. **투사체 연동**: 멀티타일 오브젝트가 투사체 경로를 차단/상호작용

## 2. 데이터 모델

### 2.1 액터/적 DB 확장 (노트태그)

```
<srpgGridW:2>          // 점유 너비 (기본 1)
<srpgGridH:2>          // 점유 높이 (기본 1)
<srpgAnchor:0,0>       // 앵커 타일 (좌상단 기준 오프셋, 기본 0,0)
<srpgUnitType:actor>   // actor | enemy | object
<srpgObjectFlags:blocking,destructible>  // 오브젝트 속성 플래그
<srpgSpriteFile:barricade_2x1>           // 커스텀 스프라이트 파일명
<srpgSpriteFolder:img/srpg>              // 스프라이트 폴더 (기본 img/characters)
```

### 2.2 SrpgUnit 클래스 확장

```javascript
// 새 프로퍼티 (SRPG_Data.js SrpgUnit 클래스)
gridW        // 점유 너비 (tiles), 기본 1
gridH        // 점유 높이 (tiles), 기본 1
anchor       // {x, y} 앵커 오프셋 (이벤트 좌표 = 점유 영역 내 앵커 위치)
unitType     // 'actor' | 'enemy' | 'object'
objectFlags  // Set<string>: 'blocking', 'destructible', 'transparent', 'cover'
spriteFile   // 커스텀 스프라이트 파일명 (null이면 기본 캐릭터 스프라이트)
spriteFolder // 스프라이트 폴더 경로

// 계산 프로퍼티
get occupiedTiles()  // [{x,y}, ...] 점유 중인 모든 타일 좌표 배열
get originX()        // 점유 영역 좌상단 X = this.x - anchor.x
get originY()        // 점유 영역 좌상단 Y = this.y - anchor.y

// 메서드
occupies(tx, ty)     // 이 유닛이 (tx,ty)를 점유하는가?
canPlaceAt(x, y)     // 이 유닛을 (x,y) 앵커로 배치할 때 모든 타일이 유효한가?
```

### 2.3 점유 영역 계산

```
유닛 앵커 좌표 = (x, y)  
점유 영역 원점 = (x - anchor.x, y - anchor.y)  
점유 타일 = { (ox+i, oy+j) | 0 ≤ i < gridW, 0 ≤ j < gridH }
```

예시: 2×3 오브젝트, anchor=(0,0), 이벤트 위치=(5,3)
```
점유: (5,3) (6,3) (5,4) (6,4) (5,5) (6,5)
     ■ ■     ← y=3
     ■ ■     ← y=4
     ■ ■     ← y=5
     x=5 x=6
```

## 3. 영향 범위 및 수정 계획

### 3.1 그리드 점유 시스템 (SRPG_Data.js)

#### isOccupied(x, y, excludeUnit) — Line 1508
**현재**: `unit.x === x && unit.y === y` 비교
**변경**: `unit.occupies(x, y)` 호출로 교체

```javascript
// Before
if (u.x === x && u.y === y) return true;

// After
if (u.occupies(x, y)) return true;
```

#### unitAt(x, y) — SRPG_SM.js Line 184
**변경**: 동일하게 `unit.occupies(x, y)` 사용

### 3.2 이동 / 경로탐색 (SRPG_Data.js)

#### calcMoveRange(unit, maxMov) — Line 1517
**현재**: BFS에서 다음 타일 (nx, ny) 1칸만 점유 확인
**변경**:
1. 이동 중인 유닛의 gridW×gridH 전체 타일이 갈 수 있는지 확인
2. `canPlaceAt(nx, ny)` → 모든 점유 타일이 passable + 다른 유닛 없음

```javascript
// 이동 가능 판정 (멀티타일)
function canMoveHere(unit, nx, ny, excludeSelf) {
    const ox = nx - unit.anchor.x;
    const oy = ny - unit.anchor.y;
    for (let i = 0; i < unit.gridW; i++) {
        for (let j = 0; j < unit.gridH; j++) {
            const tx = ox + i, ty = oy + j;
            if (!isPassable(tx, ty)) return false;
            if (isOccupied(tx, ty, excludeSelf ? unit : null)) return false;
        }
    }
    return true;
}
```

#### findPath(unit, tx, ty, moveRange) — Line 1566
**변경**: BFS 경로 탐색도 동일하게 `canMoveHere` 사용. 경로의 각 스텝에서 멀티타일 전체 배치 가능 여부 확인.

#### 이동 비용
**1×1 유닛**: 기존과 동일 (타일당 1)
**멀티타일 유닛**: 앵커 타일 기준 이동 비용 1 (점유 타일 중 최대 비용을 쓰는 방식도 고려 가능하나, 우선 단순화)

### 3.3 공격 범위 / 타겟팅 (SRPG_Data.js, SRPG_SM.js)

#### calcAtkRange(unit, fromX, fromY) — Line 1556
**변경**: 공격 범위 계산의 기준점은 앵커 타일. 멀티타일 유닛의 경우 점유 타일 중 아무 타일에서나 공격 가능하도록 확장할지, 앵커 기준만 쓸지 선택.

**권장**: 점유 타일 중 가장 가까운 타일 기준으로 사거리 계산
```
실효 거리 = min( manhattan(occupiedTile, target) for each occupiedTile )
```

#### 타겟 선택 (_handleSelectTarget)
**변경**: 커서가 멀티타일 유닛의 아무 점유 타일 위에 있으면 해당 유닛이 선택됨.
`unitAt(cursorX, cursorY)` → 자동으로 `occupies()` 기반이므로 추가 수정 불필요.

### 3.4 투사체 시스템 (SRPG_Data.js)

#### traceLine(fromX, fromY, toX, toY) — Line 1439
**현재**: Bresenham 라인 트레이싱, 각 타일에서 지형 + 오브젝트 체크
**변경**: 오브젝트 충돌 판정 시 `unitAt(x, y)`를 사용 → occupies 기반이므로 멀티타일 오브젝트 자동 감지

#### doesTileBlock(x, y, fireMode) — Line 1428
**추가 로직**: 해당 타일을 점유하는 유닛이 있고, 그 유닛의 objectFlags에 따라 차단 판정

```javascript
// 오브젝트 플래그별 투사체 상호작용
blocking      → direct & arc 모두 차단
cover         → direct만 차단 (arc는 통과)
transparent   → 투사체 통과 (시각적 장애물이지만 차단 안함)
destructible  → 차단 + 데미지 전달 (기존 srpgObject 동작)
```

### 3.5 스프라이트 렌더링 (SRPG_UI.js)

#### 현재 스프라이트 시스템
- RMMZ 기본 캐릭터 스프라이트 (48×48 기준 1타일)
- `PORTRAIT_MODE`에서 스탠딩 이미지 오버레이

#### 멀티타일 스프라이트 변경

```javascript
// 스프라이트 크기 = gridW × TILE, gridH × TILE
// 스프라이트 위치 = originX 기준 (좌상단)
sprite.width  = unit.gridW * TILE;
sprite.height = unit.gridH * TILE;
sprite.x = unit.originX * TILE;
sprite.y = unit.originY * TILE;
```

**이미지 소스 우선순위**:
1. `spriteFile`이 지정된 경우 → `spriteFolder/spriteFile.png` 로드, 전체 영역에 맞춤
2. 미지정 시 → 기본 RMMZ 캐릭터 스프라이트를 앵커 타일에 표시

**렌더링 방식**:
- 커스텀 스프라이트: 전체 점유 영역에 1:1 매핑 (이미지가 W×H 타일에 딱 맞음)
- 기본 캐릭터: 앵커 타일 위치에 일반 크기로 표시 + 나머지 타일은 반투명 점유 표시

### 3.6 오브젝트 소환 시스템 (NEW)

#### 소환 스킬 메커니즘

```
<srpgSummon:actorId>       // 소환할 액터 ID
<srpgSummonRange:1-3>      // 소환 가능 거리 (사거리 시스템 활용)
<srpgSummonDuration:3>     // 지속 턴 (0 = 영구)
<srpgSummonLimit:2>        // 동시 소환 최대 수
```

**소환 흐름**:
1. 스킬 사용 → 배치 모드 진입 (이동 범위 표시와 유사)
2. 배치 가능 타일 하이라이트 (멀티타일 배치 가능 여부 실시간 검증)
3. 타일 선택 → 해당 위치에 새 RMMZ 이벤트 동적 생성
4. 이벤트에 액터 DB 정보(gridW, gridH, sprite, objectFlags 등) 자동 적용
5. `SM._units`에 SrpgUnit으로 등록

#### 소환된 오브젝트 관리
- 소환자 참조: `summonerId` (소환자 사망/퇴장 시 동작 결정)
- 턴 카운트: `remainingTurns` (0이면 영구, 매 턴 감소 → 0에서 제거)
- 동시 제한: 소환자별 현재 소환 수 추적

## 7. 리스크 및 고려사항

### 7.1 성능
- 멀티타일 BFS는 타일당 W×H 체크 → O(N × W × H) 복잡도 증가
- 대부분 유닛은 1×1이므로 실제 영향 제한적
- 최적화: 1×1 유닛은 기존 로직 fast-path 유지

### 7.2 회전/방향
- 2×3 오브젝트를 90° 회전하면 3×2가 됨
- 현재 설계에서는 회전 미지원 (방향별 별도 스프라이트로 대응)
- 향후 확장: `srpgRotatable` 플래그 + 방향별 gridW/gridH 스왑

### 7.3 맵 경계
- 멀티타일 유닛 배치 시 맵 밖으로 점유 타일이 벗어나지 않도록 검증 필수
- `canPlaceAt()`에서 맵 경계 체크 포함

### 7.4 기존 호환성
- 노트태그가 없는 기존 유닛은 자동으로 1×1, unitType="actor/enemy"
- 기존 `<srpgObject:true>` 이벤트도 하위 호환 유지 (gridW=1, gridH=1, objectFlags=destructible)
