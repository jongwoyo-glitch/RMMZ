## 2. SRPG 전투 엔진 (SRPG_Core.js — 11,552줄)

3-소스 빌드 체계: `SRPG_Data.js` + `SRPG_SM.js` + `SRPG_UI.js` → `build.py` → `SRPG_Core.js`

### 2.1 핵심 데이터 계층 (SRPG_Data — 5,581줄)

**SrpgUnit 클래스** — 전장 위의 모든 유닛 (액터/적/오브젝트)
- RMMZ 액터/적 DB 연동: HP, MP, 능력치, 장비, 스킬, 특성(Traits)
- xparam(10종) / sparam(10종) 완전 구현: HIT, EVA, CRI, CEV, MEV, CNT, HRG, MRG, TGR, MCR, PDR, MDR, REC, PHA, FDR, EXR 등
- 멀티타일 시스템: 가변 크기(W×H) 유닛, 앵커 기반 점유, 이동/전투 대응
- 파괴 가능 오브젝트: HP 있는 사물, 장판/원소 반응 대응
- 상태이상 관리: 3티어 승급 (T1→T2→T3), 턴 감소, 중첩 불가 처리
- 버프/디버프: RMMZ 표준 8종 + 턴 관리
- 언데드/부패: 치유 반전, 독 면역/취약 분기
- HRG/MRG 턴 재생: 매 턴 시작 시 자동 회복

**SrpgSkillMeta** — 스킬 범위/투사체 메타데이터 통합
- 3-tier 범위 해석: 노트태그 → 무기 타입 기본값 → 글로벌 기본값
- 투사체 패턴: 직선(straight), 포물선(arc), 방사(radial), 관통(pierce), 체인(chain)
- 스킬→장판/구름 생성 메타: `<srpgSpawnSurface>`, `<srpgSpawnCloud>`

**SrpgSummon** — 오브젝트 소환 시스템
- 액터 DB 기반 소환 (신규) + 레거시 타입 기반 소환 (하위 호환)
- 소환물 HP/능력치, 장판 반응, 턴오더 참가

### 2.2 상태 머신 + AI (SRPG_SM — 3,420줄)

**전투 페이즈 상태 머신**
```
idle → deployment → turnAdvance → playerTurn / enemyTurn → combat → battleEnd
```

**플레이어 턴 서브 상태**
```
browse → awaitCommand → moving → radialMenu → subRadial → slotReel
  → selectTarget → combatPreview → executing
  → faceDirection (보조행동)
  → selectThrowTarget (투척)
  → selectBonusTarget (보조행동 타겟)
```

**행동 체계**
- 주행동(mainAction) + 보조행동(bonusAction) 분리
- 주행동: 공격, 스킬, 아이템 사용, 투척
- 보조행동: 방향 전환, 은신, 스프린트
- AP 기반 턴오더 (1000 AP 도달 시 행동, AGI 비례 충전)

**적 AI 시스템**
- RMMZ DB 행동 패턴 기반 스킬 선택
- 범위/사거리 기반 공격 타일 탐색
- 장판 위험도 보정 (이동/접근 경로 평가)
- 협동 공격 자동 감지 + 기회 공격 체크

**투척 시스템 (SrpgThrow)**
- 근력 기반 투척 범위
- 그리드 인벤토리 아이템 투척
- 투사체 애니메이션 분기 (직선/포물선)

### 2.3 전술 시스템 5종

| 시스템 | 내용 |
|---|---|
| **배면/측면 공격** | 4방향 판정, 물리 공격 한정, 배면 ×1.5 / 측면 ×1.25 |
| **지형 레벨(고저차)** | Region ID 기반, 높은 곳→낮은 곳 사거리+1, 저지대 회피율 보정 |

| **기회 공격** | ZoC 이탈 시 자동 공격, 이동 경로 체크, AI도 적용 |
| **협동 공격** | 인접 아군 자동 참전, 대상 인접 아군 탐색, 보너스 공격 실행 |

## 3. 그리드 인벤토리 + 관계 시스템 (GridInventory.js — 1,455줄)

### 3.1 그리드 인벤토리

- Diablo 2 스타일 2D 그리드 (기본 8×6)
- 아이템별 가변 크기 (W×H), 노트태그 `<gridSize:W,H>`
- 핫키 슬롯 (기본 4개, 소비 아이템 전용)
- 배치 ID 시스템: findItem(), removePlacement(), removeItem()
- 액터별 독립 인벤토리 + 공용 보관함

### 3.2 관계 시스템 v2 (3축)

| 축 | 설명 | 범위 |
|---|---|---|
| **Impression (인상)** | 비대칭 A→B, Bond Tag 모디파이어 자동 적용 | -100 ~ +100 |
| **Mood (기분)** | 파티원 인상 평균에서 관성(lerp 0.3) 추종 | -100 ~ +100 |
| **Bond Tag (관계)** | 이벤트 스크립트로만 전환, 히스토리 자동 기록 | 10종 |

**Bond Tag 10종**: stranger, acquaintance, friend, rival, sworn, hostile, mentor, pupil, lover, broken

**핵심 API**:
- `getImpression(id)`, `addImpression(id, amount)` — Bond Tag 승수 자동 적용
- `setBondTag(id, tag, reason)` — 히스토리 자동 기록
- `complianceRate(requesterId)` — 비전투 순응 확률 (%)
- `checkCompliance(requesterId)` — 순응 판정 실행
- `recalcMood()` — 파티원 인상 기반 기분 재계산
- `decayImpressions()` — 자연 감쇠 (|imp| > 30, Bond Tag 차단 규칙 적용)

**플러그인 커맨드 7종**: SetImpression, AddImpression, AddImpressionBi, SetBondTag, SetMood, AddMood, RecalcMood

**하위 호환**: 구 API(addFavor, getRelationship, SetRelation) 래핑, 구 세이브 자동 마이그레이션

### 3.3 퀘스트/이벤트 로그

- addQuest(), completeQuest(), updateQuest()
- addEventRecord() — 스토리 이벤트 기록 (관련 인물, 선택지 저장)

### 3.4 저장/로드

- 액터별 그리드 인벤토리, 관계 4필드, 스킬소스 개별 저장
- 구 세이브 마이그레이션: favor→impression, trust→bondTag 자동 변환
