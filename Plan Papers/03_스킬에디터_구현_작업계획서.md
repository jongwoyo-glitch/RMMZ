## Step 1: 공유 파서 모듈

**파일**: `Project1/js/plugins/src/srpgPhaseParser.js`

**역할**: 노트태그 텍스트 ↔ JSON 양방향 변환

**구현 내용**:
- `parsePhases(noteText)` → JSON 배열
  - `<srpgPhase:타입|key:val|key:val>` 형식 파싱
  - 숫자/불린/배열 자동 변환
  - 기존 태그(`<srpgReach>`, `<srpgRange>`, `<srpgArea>`, `<srpgRotate>`, `<srpgSelfTarget>`, 투사체 태그 등) 별도 파싱
- `serializePhases(jsonArray)` → 노트태그 텍스트
  - JSON → `<srpgPhase:...>` 줄 생성
- `parseReach(noteText)` → 좌표 배열
- `parseArea(noteText)` → 좌표 배열
- `serializeReach(coordArray)` → `<srpgReach:...>` 태그
- `serializeArea(coordArray)` → `<srpgArea:...>` 태그
- `mergeIntoNote(originalNote, newPhases, newReach, newArea)` → 기존 태그 보존하며 교체

**검증**: Node.js에서 단위 테스트 (기존 스킬 230~232의 노트로 파싱→직렬화→재파싱 일치 확인)

**산출물**: srpgPhaseParser.js (UMD 모듈 — 브라우저/Node 양쪽 사용 가능)

## Step 2: HTML 팝업 에디터 — 골격 + 그리드 에디터

**파일**: `Project1/tools/skill_editor.html`

**구현 내용**:
- 단일 HTML (CSS+JS 내장, 가호록 스타일)
- 다크 테마 UI 기본 레이아웃 (헤더 / 좌측 그리드 / 우측 타임라인 / 하단 패널)
- **헤더**: 스킬명 표시, 카테고리 4버튼 토글, 오행 5버튼 토글
- **그리드 에디터**:
  - Canvas 기반 타일 그리드 (5×5 ~ 11×11 가변)
  - 사거리 탭: 타일 클릭/우클릭으로 편집, 다이아몬드/십자/직선 자동생성 버튼
  - 효과범위 탭: 동일 방식, srpgRotate/srpgSelfTarget 체크박스
  - 시전자 위치(★) 고정 표시, 오행 색상 반영
- srpgPhaseParser.js 내장 (인라인 또는 script src)

**검증**: 브라우저에서 단독 열기 → 그리드 클릭으로 사거리/범위 편집 → 노트태그 텍스트 출력 확인

## Step 3: HTML 팝업 에디터 — Phase 타임라인 + 파라미터 패널

**구현 내용** (skill_editor.html 계속):
- **Phase 타임라인 (우측)**:
  - 세로 블록 리스트, 원자 타입별 아이콘+색상
  - [+ Phase 추가] 버튼 → 타입 선택 드롭다운
  - 블록 드래그로 순서 변경 (HTML5 drag & drop)
  - 블록 삭제(×) 버튼
  - 조건(if) 뱃지, 반복(×N) 뱃지 표시
- **파라미터 패널 (하단)**:
  - 블록 클릭 시 해당 원자의 파라미터 UI 동적 렌더링
  - 원자별 전용 컨트롤: 라디오 버튼, 슬라이더, 숫자 스핀, 체크박스
  - 구현할 원자 UI (이번 구현 범위):
    - 이동계: dash, push, pull, escape
    - 공격계: hit, proj, chain
    - 공통: if 조건, repeat
  - 파라미터 변경 시 즉시 JSON 갱신 → 노트태그 실시간 업데이트

**검증**: Phase 추가/삭제/순서변경 → 파라미터 수정 → 하단 노트태그 텍스트 반영 확인

## Step 4: HTML 팝업 에디터 — 프리셋 + 노트태그 입출력

**구현 내용** (skill_editor.html 계속):
- **프리셋 시스템**:
  - 카테고리(4) × 오행(5) = 20종 프리셋 데이터 내장
  - 프리셋 선택 시 phase 배열 + 사거리/범위 자동 구성
  - 프리셋은 시작점, 이후 자유 편집 가능
- **노트태그 입출력**:
  - [노트태그 보기] 버튼 → 현재 상태를 텍스트로 표시
  - [노트태그 붙여넣기] → 기존 노트태그 역파싱하여 에디터에 로드
  - [저장] → postMessage로 부모 창에 JSON 전달 (팝업 모드)
  - [저장] → Skills.json 직접 기입 (독립 실행 모드)
- **독립 실행 모드**:
  - File System Access API로 프로젝트 폴더 선택
  - Skills.json / Weapons.json 로드 → 스킬 목록 사이드바 표시
  - 스킬 선택 → 노트란 로드 → 편집 → 저장 시 JSON 기입

**검증**: 기존 스킬(230 화살사격, 231 화염탄, 232 번개광선)의 노트 역파싱 → 편집 → 재저장 → 원본 대비 검증

## Step 5: HTML 팝업 에디터 — 간이 프리뷰 시뮬레이션

**구현 내용** (skill_editor.html 계속):
- **프리뷰 그리드** (7×7):
  - 시전자(파랑 ★) + 더미 타겟(빨강 ●) 배치 (드래그 이동)
  - [▶ 실행] 클릭 시 phase 순차 애니메이션:
    - dash: 시전자 도형이 경로를 따라 이동
    - hit: 타겟 위치에 원형 이펙트 확산
    - push: 타겟 도형이 밀려남
    - pull: 타겟 도형이 끌려옴
    - chain: 타겟 간 선(arc) 연결 순차 표시
    - proj: 시전자→타겟 도형 이동
    - escape: 시전자 도형 이탈
  - 각 phase 간 0.5초 딜레이
  - 사거리/효과범위 오버레이 표시
- requestAnimationFrame 기반, Canvas 렌더링

**검증**: 돌진+타격+밀치기 조합 시뮬레이션 → 시각적으로 phase 흐름 확인

## Step 6: RMMZ 에디터 플러그인 — 버튼 + 요약 패널

**파일**: `Project1/js/plugins/SRPG_SkillEditor.js`

**구현 내용**:
- RMMZ 에디터(NW.js) 환경 감지
- 스킬/무기 편집 다이얼로그 열릴 때 DOM 감시 (MutationObserver)
- 노트란(textarea) 아래에 요약 패널 DOM 삽입:
  - Phase 흐름도: 원자 블록 가로 체인 (아이콘+텍스트, 조건 뱃지)
  - 사거리 미니맵: 인라인 Canvas (5×5 그리드)
  - 효과범위 미니맵: 인라인 Canvas
  - 요약 텍스트 한 줄
- 노트란 `input` 이벤트 감시 → 요약 패널 실시간 갱신
- [🔧 스킬 설계] 버튼 삽입 → 클릭 시 skill_editor.html 팝업 열기
- 팝업에서 postMessage 수신 → 노트란 텍스트 교체 → 요약 갱신

**주의**: RMMZ 에디터 내부 DOM 구조 파악 필요. 에디터 버전에 따라 달라질 수 있으므로 선택자를 유연하게 설계.

**검증**: RMMZ 에디터에서 스킬 편집 화면 열기 → 요약 패널 표시 확인 → 버튼 클릭 → 팝업 편집 → 저장 → 노트란 반영 확인
