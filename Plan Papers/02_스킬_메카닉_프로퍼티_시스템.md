# 스킬 메카닉 프로퍼티 시스템 설계서

> v0.1 | 2026-04-18 | 개념 설계 단계

---

## 0. 설계 철학

- **원자 조합형** — 모든 스킬은 소수의 원자 동작(Atomic Action)을 순차/조건 조합하여 구성
- **무기도 스킬이다** — 무기의 기본 공격 = phase가 1~2개인 단순 스킬. 같은 프레임워크로 처리
- **오행은 메카닉 성향** — 데미지 속성이 아니라, 어떤 원자 조합을 선호하느냐의 차이
- **노트태그 기반** — RMMZ 스킬/무기 노트란에 `<srpgPhase:...>` 태그로 정의, 데이터 드리븐

---

## 1. 원자 동작 (Atomic Actions)

### 1-1. 이동계 (Movement)

| ID | 이름 | 설명 | 주요 파라미터 |
|----|------|------|---------------|
| `dash` | 돌진 | 시전자가 대상/지점까지 이동 | path(line/arc/jump), stopAt(adjacent/on/behind), speed |
| `pull` | 끌어당김 | 대상을 시전자 방향으로 끌어옴 | dist, stopAt(adjacent/on), anim |
| `push` | 밀침 | 대상을 밀어냄 | dir(away/side_L/side_R/free), dist, casterFollow(bool) |
| `throw` | 넘기기 | 대상을 시전자 뒤로 투척 | dist, arc(bool), landDamage(bool) |
| `swap` | 위치교환 | 시전자↔대상 위치 교환 | anim |
| `escape` | 이탈 | 시전자가 지정 위치로 빠짐 | dest(back/free/random), dist |
| `leap` | 도약 | 시전자가 포물선으로 목적지 착지 | height, landAOE(bool) |

#### 이동 충돌 규칙

이동계 원자는 타일 위 유닛/오브젝트/지형과 충돌할 수 있다:

- **push/throw로 벽·오브젝트에 충돌** → 이동 중단 + 추가 충돌 데미지(옵션)
- **push로 다른 유닛에 충돌** → 연쇄 밀침(옵션) 또는 양쪽 모두 데미지
- **dash 경로 위 장애물** → 장애물 앞에서 정지(기본) 또는 관통(옵션)
- **pull로 끌다가 장애물** → 장애물 위치에서 정지

```
충돌 파라미터:
  onCollide: "stop" | "damage" | "chain_push" | "pierce"
  collideDamage: 0.5  (원래 데미지의 비율)
```

### 1-2. 공격계 (Offense)

| ID | 이름 | 설명 | 주요 파라미터 |
|----|------|------|---------------|
| `hit` | 단타 | 단일/범위 즉시 타격 | area(오프셋 배열), anim, element |
| `proj` | 투사체 | 기존 3종(projectile/hitray/artillery) | (기존 투사체 시스템 파라미터 전부) |
| `chain` | 전파 | 타겟→주변 타겟으로 연쇄 전파 | maxBounce, searchRadius, damageDecay, splitCount, canRepeat, targetTeam, condition |
| `multi` | 다중지정 | N개 타겟을 개별 선택 후 각각 처리 | count, perTarget(서브 phase 배열), sharePool(bool) |
| `dot` | 지속피해 | 대상에 턴당 데미지 부여 | damage, duration, stackable(bool) |

#### chain 상세 — 전파 패턴 변형

```
chain 기본형 (체인 라이트닝):
  maxBounce: 4, searchRadius: 3, damageDecay: 0.8
  splitCount: 1, canRepeat: false, targetTeam: "enemy"

감쇠형 (기본): damageDecay < 1.0 → 바운스마다 데미지 ↓
증폭형 (수·얼음): damageDecay > 1.0 → 바운스마다 데미지 ↑ (중첩)
분기형 (번개 갈래): splitCount: 2 → 바운스마다 2갈래 분기
조건부 전파: condition: "state:둔화" → 해당 상태 걸린 적에게만 전파
아군 전파 (체인 힐): targetTeam: "ally", damageDecay: 0.9
```

#### multi 상세 — 다중 타겟 지정

```
스톡 샷 (투사체 3발):
  count: 3, sharePool: false
  perTarget: [{ type: "proj", projType: "projectile" }]

  → UI: 타겟 선택을 3회 수행, 같은 적 중복 가능 여부는 sharePool로 제어
  → sharePool: true → 같은 적 중복 지정 가능 (한 놈에 3발 몰아치기)
  → sharePool: false → 서로 다른 적만 지정
```

### 1-3. 설치계 (Placement)

| ID | 이름 | 설명 | 주요 파라미터 |
|----|------|------|---------------|
| `summon` | 소환 | 기존 소환 시스템 | (기존 파라미터) |
| `terrain` | 지형변형 | 타일의 지형 태그/효과를 변경 | area, newTag, effect, duration |
| `trap` | 함정 | 적 진입 시 발동하는 숨겨진 설치물 | trigger(enter/endTurn), onTrigger(서브 phase), visible(bool), duration |

#### terrain 상세 — 동적 지형

```
현재 지형 태그 (정적):
  0=일반, 1=낭떠러지, 2=엄폐물, 3=폐쇄벽

동적 지형 효과 (terrain 원자로 생성):
  fire_field    — 진입/턴 종료 시 화상 데미지
  mud           — 이동력 -2, 회피 -20%
  ice           — 이동 시 미끄러짐 (관성 1칸 추가이동)
  thorns        — 진입 시 고정 데미지
  heal_zone     — 턴 시작 시 HP 회복
  smoke         — 원격 공격 명중률 ↓, 은신 유지 가능

terrain 파라미터:
  area: [{dx:0,dy:0},{dx:1,dy:0}...]  (효과 범위)
  effect: "mud"
  duration: 3  (턴 후 소멸, 0=영구)
  teamFilter: "enemy" | "ally" | "all"
```

### 1-4. 반응계 (Reaction)

| ID | 이름 | 설명 | 주요 파라미터 |
|----|------|------|---------------|
| `counter` | 카운터 | 피격 시 자동 반격 (기존 확장) | condition, skill, chance |
| `interrupt` | 차단 | 적 스킬 시전을 방해 | range, targetPhase(casting/channel), chance |
| `reflect` | 반사 | 투사체/마법을 되돌려보냄 | projOnly(bool), damageRatio |
| `trigger` | 조건발동 | 특정 조건 충족 시 자동 실행 | on(allyHit/enemyMove/hpBelow), response(서브 phase) |

> 반응계는 스킬의 phase에 들어가지 않고, **패시브 프로퍼티**로 유닛에 부여된다.
> 스킬 노트태그가 아닌 스킬/장비의 특성(trait) 시스템으로 관리.

## 2. Phase 파이프라인

### 2-1. 기본 구조

하나의 스킬 = **phase 배열**의 순차 실행:

```
skill.phases = [
  { type: "dash", ... },     // phase 0: 돌진
  { type: "hit",  ... },     // phase 1: 타격
  { type: "push", ... },     // phase 2: 밀침
]
```

각 phase는 **앞 phase의 결과 컨텍스트**를 받는다:

```
context = {
  casterPos: {x, y},      // 시전자 현재 위치 (dash 이후 변동됨)
  targetPos: {x, y},      // 주 타겟 현재 위치 (push 이후 변동됨)
  targets: [...],          // 영향받은 유닛 목록
  lastDamage: Number,      // 직전 phase의 데미지
  isCrit: Boolean,         // 직전 phase의 크리티컬 여부
  phaseIndex: Number       // 현재 몇 번째 phase
}
```

### 2-2. 조건부 Phase (Conditional)

phase에 `if` 필드를 붙이면 조건 충족 시에만 실행:

```
{ type: "hit", area: [...], if: "crit" }           // 크리티컬일 때만 추가타
{ type: "push", dist: 2, if: "kill" }              // 처치 시 시체 밀침 (재배치)
{ type: "dash", dest: "free", if: "hpBelow:30%" }  // HP 30% 미만 시 자동 이탈
```

지원 조건:
- `crit` — 직전 phase가 크리티컬
- `kill` — 직전 phase로 대상 처치
- `hit` — 직전 phase가 명중 (미스 아님)
- `miss` — 직전 phase가 미스
- `hpBelow:N%` — 시전자 HP가 N% 미만
- `targetState:상태명` — 대상이 특정 상태이상 보유
- `allyNearby:N` — 반경 N 내 아군 존재
- `elementMatch:오행` — 시전자의 주 오행이 일치

### 2-3. 반복 Phase (Loop)

```
{ type: "hit", repeat: 3, interval: 10 }  // 10프레임 간격으로 3회 타격
{ type: "chain", ..., repeat: "untilMiss" } // 미스 날 때까지 체인
```

### 2-4. 병렬 Phase (Parallel)

기본은 순차(sequential)이지만, 동시 실행이 필요한 경우:

```
{ parallel: [
  { type: "proj", target: "A", ... },
  { type: "proj", target: "B", ... },
  { type: "proj", target: "C", ... }
]}
```

multi 타입의 내부 처리가 이 병렬 phase로 풀림.

## 3. 노트태그 설계

### 3-1. 기본 문법

```
<srpgPhase:타입|파라미터1:값|파라미터2:값>
```

### 3-2. 예시 — 대표 스킬 패턴

**돌진 베기 (목-물리근접)**
```
<srpgPhase:dash|path:line|stopAt:adjacent>
<srpgPhase:hit|anim:12>
```

**돌진 후 광역 (화-물리근접)**
```
<srpgPhase:dash|path:line|stopAt:adjacent>
<srpgPhase:hit|area:cross3|anim:24>
```

**방패 밀치기 (토-물리근접)**
```
<srpgPhase:hit|anim:8>
<srpgPhase:push|dir:away|dist:2|onCollide:damage>
```

**옆으로 밀어내기 (토-물리근접 변형)**
```
<srpgPhase:hit|anim:8>
<srpgPhase:push|dir:side_L|dist:2>
```

**밀치고 따라가기 (토 변형)**
```
<srpgPhase:hit|anim:8>
<srpgPhase:push|dir:away|dist:3|casterFollow:true>
```

**뒤잡기 (수-물리근접)**
```
<srpgPhase:dash|path:arc|stopAt:behind>
<srpgPhase:hit|anim:15>
```

**카운터 일격 (금-물리근접)**
```
<srpgPhase:hit|anim:20>
<srpgPhase:hit|if:crit|anim:21|bonusDamage:0.5>
```

**체인 라이트닝 (목-마법공격)**
```
<srpgPhase:proj|type:hitray>
<srpgPhase:chain|maxBounce:4|radius:3|decay:0.8|proj:lightning_arc>
```

**얼음 중첩 전파 (수-마법공격)**
```
<srpgPhase:proj|type:projectile>
<srpgPhase:chain|maxBounce:3|radius:3|decay:1.15|condition:state:둔화>
```

**스톡 샷 (물리원격 다중)**
```
<srpgPhase:multi|count:3|sharePool:true>
<srpgPhase:proj|type:projectile|perTarget:true>
```

**위기탈출 (수-물리근접)**
```
<srpgPhase:hit|area:self_radius1|anim:30>
<srpgPhase:escape|dest:back|dist:3>
```

**끌어당기기 (수-마법보조)**
```
<srpgPhase:pull|dist:99|stopAt:adjacent>
```

**넘기기 (수 변형)**
```
<srpgPhase:pull|dist:99|stopAt:on>
<srpgPhase:throw|dir:behind|dist:2|landDamage:true>
```

**진흙탕 생성 (토-마법보조)**
```
<srpgPhase:terrain|area:radius2|effect:mud|duration:3>
```

**함정 설치 (수-물리원격)**
```
<srpgPhase:trap|trigger:enter|visible:false|duration:5>
<srpgPhase:hit|anim:40|perTrap:true>
<srpgPhase:push|dir:away|dist:1|perTrap:true>
```

**포격 후 화염지대 (화-마법공격)**
```
<srpgPhase:proj|type:artillery>
<srpgPhase:hit|area:radius2|anim:24>
<srpgPhase:terrain|area:radius2|effect:fire_field|duration:2>
```

**체인 힐 (화-마법보조)**
```
<srpgPhase:hit|targetTeam:ally|heal:true|anim:50>
<srpgPhase:chain|maxBounce:2|radius:3|decay:0.9|targetTeam:ally|heal:true>
```

## 4. 오행별 메카닉 성향 매트릭스

### 4-1. 선호 원자 패턴

| 오행 | 핵심 키워드 | 주력 원자 | 보조 원자 | 기동력 |
|------|------------|-----------|-----------|--------|
| 목(木) | 기동·연타 | dash, hit×N, chain | proj(hitray) | 빠름 |
| 화(火) | 화력·범위 | hit(AOE), proj(artillery), terrain(fire) | chain(heal) | 보통 |
| 토(土) | 진지·설치 | push, summon, terrain(mud/barrier) | hit(협동) | 느림 |
| 금(金) | 반응·정밀 | counter, hit(if:crit), interrupt | proj(long range) | 보통 |
| 수(水) | 제어·교란 | pull, throw, escape, trap | chain(조건부), terrain(ice) | 빠름 |

### 4-2. 전투카테고리별 상세 (1.xlsx 매핑 + 메카닉 연결)

#### 물리-근접

| 오행 | 무기 | 대표 패턴 | phase 구성 |
|------|------|-----------|-----------|
| 목 | 맨손, 한손, 쌍수 | 속공 연타, 돌진 | dash→hit×N |
| 화 | 양손, 대형 | 대범위 강타 | dash→hit(AOE) |
| 토 | 한손+방패 | 밀치기, 협동 | hit→push, hit(allyNearby 보정) |
| 금 | 양손, 폴암 | 카운터, 중거리 | hit(if:crit→추가타), counter(패시브) |
| 수 | 한손, 단검 | 은신, 뒤잡기 | dash(arc,behind)→hit→escape |

#### 물리-원격

| 오행 | 무기 | 대표 패턴 | phase 구성 |
|------|------|-----------|-----------|
| 목 | 직사화기 | 연발 사격 | proj×N(repeat) |
| 화 | 직사화기 | 범위 사격 | proj→hit(AOE) |
| 토 | 손쇠뇌, 쌍수 | 터렛·엄폐물 | summon(turret), summon(cover) |
| 금 | 곡사+직사 | 초장거리 저격 | proj(artillery, 긴 사거리, 고크리뎀) |
| 수 | 한손 손쇠뇌 | 속성볼트, 함정 | proj(속성), trap 설치 |

#### 마법-공격

| 오행 | 마법 | 대표 패턴 | phase 구성 |
|------|------|-----------|-----------|
| 목 | 번개 | 관통, 다단히트 | proj(hitray, piercing)→chain(분기) |
| 화 | 화염 | 누킹, 방사형 | proj(artillery)→hit(AOE)→terrain(fire) |
| 토 | 대지 | CC+도트 | hit(AOE,small)→dot→terrain(mud) |
| 금 | 바람 | 넓은범위, 크리 | hit(AOE,large, 저뎀 고크리) |
| 수 | 물/얼음 | 턴넘김CC, 중첩 | hit→chain(decay>1, 조건부)→dot(stack) |

#### 마법-보조

| 오행 | 역할 | 대표 패턴 | phase 구성 |
|------|------|-----------|-----------|
| 목 | 소환 | 정령 소환·강화 | summon(정령)→buff(소환수) |
| 화 | 회복 | 힐, 부활 | hit(heal)→chain(heal, 아군전파) |
| 토 | 전투버프 | 공방버프, 배리어 | buff(아군)→summon(barrier)→terrain |
| 금 | 카운터 | 디스펠, 반격부여 | interrupt→reflect→dispel |
| 수 | 디버프 | 강제이동, 상태이상 | pull/push(적·아군)→debuff→terrain(ice) |

## 5. 무기 = 단순 스킬

무기의 기본공격은 phase가 1~2개인 단순 스킬로 처리:

```
장검 (물리근접-기본):
  <srpgPhase:hit|anim:1>

창 (물리근접-금, 중거리):
  <srpgPhase:hit|anim:3|reach:2>

활 (물리원격-기본):
  <srpgPhase:proj|type:projectile|anim:5>

방패 (물리근접-토, 밀치기):
  <srpgPhase:hit|anim:8>
  <srpgPhase:push|dir:away|dist:1>

단검 (물리근접-수, 이탈):
  <srpgPhase:hit|anim:2>
  <srpgPhase:escape|dest:back|dist:1|if:kill>
```

무기에 기본 내장된 phase + 스킬이 추가하는 phase = 최종 phase 배열.
스킬이 `<srpgOverrideWeapon>` 태그를 가지면 무기 phase를 무시하고 스킬 자체 phase만 사용.

## 6. 기존 시스템과의 통합

### 6-1. 현재 투사체 시스템 → proj 원자로 흡수

기존 `<srpgProjImage>`, `<srpgProjSpeed>` 등의 노트태그는 그대로 유지.
새 시스템에서 `{ type: "proj" }` phase가 기존 투사체 파라미터를 자동 참조.

### 6-2. 현재 소환 시스템 → summon 원자로 흡수

기존 `<srpgSummon:타입>` 노트태그도 유지.
새 시스템의 `{ type: "summon" }` phase가 기존 소환 로직을 호출.

### 6-3. 현재 반격 시스템 → counter/trigger로 확장

기존 반격(사거리 내 자동) 로직은 기본 counter로 유지.
금 계열 패시브 스킬이 counter 조건/확률/스킬을 오버라이드.

### 6-4. 현재 보조행동 → escape/dash/push와 연계

현재 스켈레톤인 밀치기/이탈 보조행동을 push/escape 원자로 구현 가능.
보조행동 = phase가 1개인 스킬 + `<srpgBonusAction>` 태그.

## 7. 스킬 에디터 UI

### 7-0. 개요

스킬/무기의 phase 파이프라인을 시각적으로 조립하는 에디터.
RMMZ 스튜디오에 통합되어 원버튼으로 팝업 에디터를 열고, 편집 결과가 스튜디오에 즉시 반영된다.

```
┌─ RMMZ 에디터 (스킬/무기 편집 화면) ─────────────────────┐
│                                                          │
│  [기존 스킬 필드들: 이름, 타입, 소모, 공식 등]           │
│                                                          │
│  ┌─ 요약 패널 (읽기 전용, 항상 표시) ──────────────────┐ │
│  │ 사거리: ◆◆◆ (다이아3)  │  효과범위: ╋ (십자3)      │ │
│  │ [돌진→인접] → [타격(AOE)] → [밀침(후방2)]           │ │
│  │ 조건: 크리티컬 시 추가타 │ 기동: 빠름               │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                          │
│  [ 🔧 스킬 설계 ]  ← 클릭하면 팝업 에디터 열림          │
│                                                          │
│  ┌─ 노트란 ───────────────────────────────────────────┐ │
│  │ <srpgPhase:dash|path:line|stopAt:adjacent>          │ │
│  │ <srpgPhase:hit|area:cross3|anim:24>                 │ │
│  │ <srpgPhase:push|dir:away|dist:2|if:crit>            │ │
│  └─────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘

#### 데이터 흐름

```
[RMMZ 노트란 텍스트]
    ↓ 파서(C)
[JSON phase 배열]
    ↓
[요약 패널(A) 렌더링]
    
사용자가 "스킬 설계" 클릭
    ↓
[팝업(B) 열림, JSON 전달]
    ↓ 편집

    ↓ 직렬화(C)
[노트태그 텍스트 → RMMZ 노트란에 기입]

#### 통신 방식

RMMZ 에디터는 NW.js 환경이므로 팝업 창과의 통신은:
- `window.open()` 으로 팝업 생성
- `window.opener.postMessage()` / `window.addEventListener('message')` 로 양방향 통신
- 팝업에서 저장 시 JSON을 postMessage로 부모에게 전달
- 부모(에디터 플러그인)가 직렬화 후 노트란에 반영

### 7-2. 레이어 A — RMMZ 에디터 요약 패널

노트란의 `<srpgPhase:...>` 태그를 실시간 파싱하여, 스킬 편집 화면 내에 읽기 전용 시각 요약을 렌더링한다.

#### 표시 항목

**Phase 흐름도 (가로 체인)**
```
[돌진(직선→인접)] ──→ [타격(십자3)] ──→ [밀침(후방2)]
       │                                    │
       빠름                              if: 크리티컬
```
- 각 블록은 원자 타입별 색상: 이동=초록, 공격=빨강, 설치=노랑, 반응=파랑
- 조건부 phase는 블록 아래에 점선 분기로 표시
- 반복 phase는 블록 위에 ×N 뱃지

**사거리 미니맵 (5×5~9×9 그리드)**
```
  · · ◆ · ·
  · ◆ ◆ ◆ ·
  ◆ ◆ ★ ◆ ◆    ★=시전자, ◆=사거리 타일
  · ◆ ◆ ◆ ·
  · · ◆ · ·
```
- 기존 `<srpgReach>` 및 `<srpgRange>` 파싱
- 시전자 위치 = 중앙 ★
- 사거리 타일 = 색상 채움

**효과범위 미니맵**
```
  · ╋ ·
  ╋ ╋ ╋    ╋=효과 타일
  · ╋ ·
```
- `<srpgArea>` 파싱
- 타겟 위치 기준 표시

**요약 텍스트 한 줄**
```
물리근접 · 목(木) · 사거리 3 · 효과범위 십자3 · 기동 빠름
```

#### 렌더링 방식

RMMZ 에디터의 스킬 편집 화면 DOM에 커스텀 `<div>` 삽입.
Canvas 또는 인라인 SVG로 그리드 미니맵 렌더링.
노트란 `textarea`의 `input` 이벤트를 감시하여 실시간 갱신.

```
진입 시점:
  RMMZ 에디터가 스킬/무기 편집 화면을 열 때
  → 플러그인이 DOM MutationObserver로 감지
  → 요약 패널 DOM 삽입
  → 노트란 텍스트 파싱 → 렌더링
  
갱신 시점:
  1. 노트란 텍스트 변경 (수동 편집 또는 팝업 저장)
  2. 다른 스킬/무기로 선택 변경
  3. 팝업 에디터에서 postMessage 수신
```

#### 7-3-1. 헤더 영역

| 항목 | UI | 비고 |
|------|-----|------|
| 스킬명 | 텍스트 표시 (편집 불가) | RMMZ에서 전달받은 스킬명 |
| 카테고리 | 4버튼 토글: 물리근접 / 물리원격 / 마법공격 / 마법보조 | 선택 시 프리셋 추천 변경 |
| 오행 | 5버튼 토글: 목 / 화 / 토 / 금 / 수 (오행 색상) | 선택 시 프리셋 추천 변경 |
| 프리셋 | 드롭다운: 카테고리×오행 조합별 대표 패턴 | 선택 시 phase 자동 구성 |

프리셋 예시:
```
물리근접-목: "속공 돌진" → dash(line,adjacent) + hit(repeat:3)
물리근접-토: "방패 밀치기" → hit + push(away,2)
마법공격-목: "체인 라이트닝" → proj(hitray) + chain(4,3,0.8)
마법보조-수: "끌어당기기" → pull(99,adjacent)
```

프리셋은 시작점일 뿐, 이후 자유롭게 수정 가능.

#### 7-3-2. 그리드 에디터 (좌측)

두 개의 탭: **사거리(Reach)** 탭과 **효과범위(Area)** 탭.

**사거리 탭:**
- 중앙 ★ = 시전자 위치 (고정)
- 타일 클릭 = 사거리 추가 (색상 채움)
- 타일 우클릭 = 사거리 제거
- 자동 생성 버튼:
  - "다이아몬드 N" — 반지름 N 맨해튼 거리 채움
  - "십자 N" — 상하좌우 N칸
  - "직선 N" — 전방 N칸
  - "전방위 N" — N칸 정사각형
- 그리드 크기 조절: 5×5 / 7×7 / 9×9 / 11×11
- 결과: `<srpgReach:dx,dy|dx,dy|...>` 또는 `<srpgRange:N>`

**효과범위 탭:**
- 중앙 ★ = 타겟/착탄 위치 (고정)
- 타일 클릭/우클릭으로 범위 편집
- 자동 생성 버튼: 다이아몬드, 십자, 3×3, 5×5
- `<srpgRotate>` 체크박스 — 공격 방향에 따라 범위 회전 여부
- `<srpgSelfTarget>` 체크박스 — 자기 위치 기준 여부
- 결과: `<srpgArea:dx,dy|...>`

**시각 표현:**
```
사거리 타일 색상:
  기본: 해당 오행 색상 (반투명)
  hover: 밝게 하이라이트
  
효과범위 타일 색상:
  기본: 주황 (반투명)
  중첩(사거리+효과): 오행색 + 주황 블렌딩
```

#### 7-3-3. Phase 타임라인 (우측)

세로 방향 블록 리스트. 각 블록은 하나의 phase를 나타냄.

**블록 구조:**
```
┌─ Phase N ────────────────────────┐
│  [아이콘] [타입명]    [×] 삭제   │
│  요약 텍스트 (1줄)               │
│  ┌─ 뱃지들 ─────────────────┐   │
│  │ if:crit │ ×3 │ decay:0.8 │   │
│  └───────────────────────────┘   │
└──────────────────────────────────┘
```

**타입별 아이콘+색상:**

| 계열 | 원자 | 아이콘 | 블록 색상 |
|------|------|--------|-----------|
| 이동 | dash | 💨 | 초록 계열 (#3a5) |
| 이동 | pull | 🧲 | 초록 계열 |
| 이동 | push | 🫸 | 초록 계열 |
| 이동 | throw | ↗️ | 초록 계열 |
| 이동 | swap | 🔄 | 초록 계열 |
| 이동 | escape | 🏃 | 초록 계열 |
| 이동 | leap | ⬆️ | 초록 계열 |
| 공격 | hit | ⚔️ | 빨강 계열 (#c44) |
| 공격 | proj | 🎯 | 빨강 계열 |
| 공격 | chain | ⚡ | 빨강 계열 |
| 공격 | multi | 🎯×N | 빨강 계열 |
| 공격 | dot | ☠️ | 빨강 계열 |
| 설치 | summon | 📦 | 노랑 계열 (#b89) |
| 설치 | terrain | 🌍 | 노랑 계열 |
| 설치 | trap | ⚠️ | 노랑 계열 |

**조작:**
- 블록 드래그 → 순서 변경
- 블록 클릭 → 하단 파라미터 패널에 상세 표시
- [+ Phase 추가] 버튼 → 원자 타입 선택 드롭다운
- [×] 버튼 → 해당 phase 삭제

**조건부/반복 표시:**
- `if` 조건이 있는 블록은 점선 테두리 + 조건 뱃지
- `repeat` 이 있는 블록은 ×N 뱃지
- `parallel` 그룹은 블록들이 가로로 나란히 배치

#### 7-3-4. 파라미터 패널 (하단)

Phase 타임라인에서 블록을 선택하면, 하단에 해당 원자의 상세 파라미터가 표시된다.

**원자별 파라미터 UI:**

**dash:**
| 파라미터 | UI 타입 | 선택지 |
|----------|---------|--------|
| 경로(path) | 라디오 버튼 | 직선 / 포물선 / 점프 |
| 도착(stopAt) | 라디오 버튼 | 대상 앞(adjacent) / 대상 위(on) / 대상 뒤(behind) |
| 속도(speed) | 슬라이더 | 1~10 (기본 5) |
| 충돌(onCollide) | 라디오 버튼 | 정지 / 관통 / 데미지 |
| 관통 데미지(collideDamage) | 슬라이더 | 0~1.0 (충돌=데미지일 때만 표시) |

**push:**
| 파라미터 | UI 타입 | 선택지 |
|----------|---------|--------|
| 방향(dir) | 라디오 버튼 + 8방향 로즈 | 후방 / 좌측 / 우측 / 자유지정 |
| 거리(dist) | 숫자 스핀 | 1~9 |
| 시전자 추적(casterFollow) | 체크박스 | — |
| 충돌(onCollide) | 라디오 버튼 | 정지 / 데미지 / 연쇄밀침 |

**hit:**
| 파라미터 | UI 타입 | 선택지 |
|----------|---------|--------|
| 효과범위(area) | 그리드 에디터 연동 | 좌측 그리드 "효과범위" 탭 활성화 |
| 애니메이션(anim) | 드롭다운 | RMMZ 애니메이션 목록 |
| 대상팀(targetTeam) | 라디오 | 적 / 아군 / 전체 |
| 힐 여부(heal) | 체크박스 | — |
| 보너스 데미지(bonusDamage) | 슬라이더 | 0~2.0 |

**chain:**
| 파라미터 | UI 타입 | 선택지 |
|----------|---------|--------|
| 최대 바운스(maxBounce) | 숫자 스핀 | 1~10 |
| 탐색 반경(searchRadius) | 숫자 스핀 | 1~9 |
| 감쇠율(damageDecay) | 슬라이더 | 0.5~1.5 (1.0 미만=감쇠, 이상=증폭) |
| 분기 수(splitCount) | 숫자 스핀 | 1~3 |
| 중복 타격(canRepeat) | 체크박스 | — |
| 대상팀(targetTeam) | 라디오 | 적 / 아군 |
| 전파 조건(condition) | 드롭다운 + 텍스트 | 없음 / 상태이상 지정 |
| 전파 투사체(proj) | 텍스트 입력 | 이미지명 |

**proj:**
| 파라미터 | UI 타입 | 선택지 |
|----------|---------|--------|
| 투사체 종류 | 라디오 | projectile / hitray / artillery |
| (이하 기존 투사체 파라미터 전부) | | |

**공통 파라미터 (모든 원자):**
| 파라미터 | UI 타입 | 선택지 |
|----------|---------|--------|
| 조건(if) | 드롭다운 | 없음/crit/kill/hit/miss/hpBelow/targetState/allyNearby |

#### 7-3-5. 프리뷰 영역

**간이 시뮬레이션:**
- 7×7 그리드에 시전자(파랑 ★)와 더미 타겟(빨강 ●) 1~3개를 배치
- [▶ 실행] 클릭 시 phase를 순차 애니메이션으로 재생
  - dash → 시전자 스프라이트 이동
  - hit → 타겟 위치에 원형 이펙트
  - push → 타겟 스프라이트 밀려남
  - chain → 타겟 간 선(arc) 연결 애니메이션
  - proj → 간이 투사체 이동
- 애니메이션은 실제 게임 수준이 아닌 간략한 도형 표현
- 더미 타겟은 드래그로 배치 변경 가능

**노트태그 보기:**
- 현재 편집 상태를 노트태그 텍스트로 변환하여 표시
- 읽기 전용 텍스트 박스
- [복사] 버튼 (수동 복사 필요 시)

### 7-4. 레이어 C — 공유 파서 (srpgPhaseParser.js)

레이어 A(요약 패널)와 레이어 B(팝업 에디터) 양쪽에서 사용하는 파싱/직렬화 모듈.

#### 파싱 (텍스트 → JSON)

```javascript
입력: "<srpgPhase:dash|path:line|stopAt:adjacent>\n<srpgPhase:hit|anim:12>"

출력: [
  { type: "dash", path: "line", stopAt: "adjacent" },
  { type: "hit", anim: 12 }
]
```

파싱 규칙:
- `<srpgPhase:` 로 시작하는 줄을 수집
- `|` 로 분리, 첫 토큰 = type, 나머지 = key:value 쌍
- value가 숫자면 Number 변환
- value가 "true"/"false"면 Boolean 변환
- value에 콤마가 있으면 배열로 변환

#### 직렬화 (JSON → 텍스트)

```javascript
입력: [
  { type: "dash", path: "line", stopAt: "adjacent" },
  { type: "hit", anim: 12, if: "crit" }
]

출력:
"<srpgPhase:dash|path:line|stopAt:adjacent>
<srpgPhase:hit|anim:12|if:crit>"
```

#### 기존 태그와의 공존

노트란에는 `<srpgPhase:...>` 이외에도 기존 태그들이 있을 수 있다:
- `<srpgReach:...>`, `<srpgRange:N>`, `<srpgArea:...>`
- `<srpgProjImage:...>`, `<srpgProjSpeed:...>` 등

파서는:
- `<srpgPhase:...>` 태그만 파싱/수정
- 다른 태그는 그대로 보존
- 사거리/효과범위 태그도 별도로 파싱하여 그리드 에디터에 표시
- 저장 시 수정된 태그만 교체, 나머지는 원본 유지

### 7-5. 무기 편집 모드

무기도 스킬과 동일한 에디터를 사용하되, 다음이 다르다:

- 헤더에 "무기 기본공격" 표시
- 프리셋이 무기 종류별로 제공 (장검, 창, 활, 방패, 단검 등)
- Phase 수가 1~2개로 제한 권장 (경고 표시, 강제는 아님)
- 팝업 내에 "이 무기를 장착한 스킬의 최종 phase" 미리보기 제공:
  ```
  무기 phase: [타격]
  + 스킬 "돌진베기" phase: [돌진] → [타격]
  = 최종: [돌진] → [타격(무기)] → [타격(스킬)]
    또는 스킬이 <srpgOverrideWeapon> 이면: [돌진] → [타격(스킬)]
  ```
