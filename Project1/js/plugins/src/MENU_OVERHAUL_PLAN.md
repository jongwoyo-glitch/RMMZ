# 메인메뉴 오버홀 구현 계획서

> 작성: 2026-04-21 | 대상: RMMZ Project1 인게임 ESC 메뉴

---

## 0. 설계 원칙

- RMMZ 네이티브 Scene_Menu를 **완전 교체** (AltMenuScreen.js 비활성화)
- 신규 플러그인 `MenuOverhaul.js`로 독립 관리
- GahoSystem.js API를 적극 활용 (원국/궁합/성향 데이터)
- SRPG_Core.js의 SrpgUnit 데이터와 연동 (전투 스탯)
- 마우스+키보드 양쪽 조작 지원

---

## 1. 메뉴 구조 최종안

```
[ESC] → Scene_CustomMenu
 ├── 상단: 파티 디스플레이 (6인 초상화 + 이름/HP/MP)
 ├── 인물 (Character)   ── 탭: 능력치 | 장비 | 프로필
 ├── 관계 (Bonds)       ── 호감도 + 원국 궁합
 ├── 소지품 (Items)     ── 그리드 인벤토리 + 핫키 슬롯
 ├── 전술 (Tactics)     ── 파티 전술 분석 + 스킬 전체목록
 ├── 일지 (Journal)     ── 퀘스트/이벤트 기록
 └── 시스템 (System)    ── 저장/불러오기/옵션
```

---

## 2. 단계별 구현 계획

### Phase 1: 백엔드 데이터 계층
> 인게임 데이터 구조 + 저장/로드 시스템

#### 1-A: 그리드 인벤토리 데이터 모델
- **Game_GridInventory** 클래스 신규 생성
  - `_grid[rows][cols]`: 2D 배열 (캐릭터별 기본 크기: 8x6)
  - `_hotkeys[N]`: 핫키 슬롯 배열 (캐릭터당 4칸, 1x1 전용)
  - `place(itemId, gx, gy)`: 아이템 배치 (충돌 검사)
  - `remove(gx, gy)`: 아이템 제거
  - `moveItem(fromX, fromY, toX, toY)`: 아이템 이동
  - `setHotkey(slotIdx, itemId)`: 핫키 등록
  - `canPlace(itemId, gx, gy)`: 배치 가능 여부
  - `getItemAt(gx, gy)`: 해당 칸의 아이템 조회
  - `getAllItems()`: 인벤토리 내 전체 아이템 목록
- 아이템 회전 없음 — 고정 형태만

#### 1-B: 아이템 부피(Size) 노트태그
- `<gridSize:W,H>` — 아이템/무기/방어구에 부피 지정
  - 미지정 시 기본값: 소비아이템 1x1, 무기 1x2, 방어구 2x2
- `<gridIcon:x,y,w,h>` — 그리드 내 표시용 아이콘 영역 (선택)
- $dataItems / $dataWeapons / $dataArmors 노트 파싱

#### 1-C: Game_Actor 확장
- `actor._gridInventory`: Game_GridInventory 인스턴스
- `actor._relationships`: 관계 데이터 (호감도/신뢰도 맵)
- `actor._learnableSkills`: 습득 가능 스킬 목록
- `actor._skillSource`: 각 스킬의 습득 경로 기록 {skillId → teacherActorId}

#### 1-D: Game_Party 확장
- `party._questLog`: 퀘스트 데이터 배열
- `party._eventLog`: 이벤트 기록 배열
- `party._sharedStorage`: 공용 보관함 Game_GridInventory (별도 접근)

#### 1-E: 저장/로드 연동
- DataManager.makeSaveContents / extractSaveContents 확장
- 그리드 인벤토리, 관계 데이터, 퀘스트/이벤트 로그 직렬화

**산출물**: 백엔드 데이터 클래스들, 노트태그 파서, 세이브/로드 연동

### Phase 2: 프론트엔드 — 메뉴 프레임워크
> Scene + Window 뼈대 구축

#### 2-A: Scene_CustomMenu 기본 골격
- Scene_MenuBase 상속
- 상단 파티 디스플레이 Window
  - 6인 스탠딩 초상화 (StandingManager 연동)
  - 이름 + HP/MP바
  - 선택된 캐릭터 강조 (위로 올라감 + 테두리)
- 좌측 또는 하단 메뉴 커맨드 Window (6개 항목)
- 우측 메인 콘텐츠 영역 (탭별 교체)

#### 2-B: Window_MenuCommand 커스텀
- 커맨드: 인물, 관계, 소지품, 전술, 일지, 시스템
- 아이콘 + 텍스트 조합
- 키보드(상하) + 마우스 클릭 지원

**산출물**: 빈 Scene_CustomMenu + 파티 디스플레이 + 커맨드 창

### Phase 3: 프론트엔드 — 인물 (Character)
> 능력치 레이더 차트 + 장비 + 프로필

#### 3-A: 능력치 탭 — 레이더 차트
- **전투 능력치 다각형**: ATK, DEF, MAT, MDF, AGI, LUK
  - SrpgUnit 또는 Game_Actor 파라미터 기반
  - PIXI.Graphics로 다각형 직접 그리기
  - 배경 눈금선(최대치 대비 비율) + 채움 영역
- **비전투 능력치 다각형**: GahoSystem 축 기반
  - PARAM_AXES 5축: 주체성, 사교성, 현실감각, 자기절제, 계획성
  - 또는 ELEM_AXES 5축: 공감능력, 몰입능력, 정서안정, 판단력, 학습능력
  - 탭 토글 또는 좌우 전환으로 두 차트 전환
- 차트 아래 수치 상세 텍스트

#### 3-B: 장비 탭
- 기존 RMMZ 장비 인터페이스 리디자인
- 좌측: 부위별 슬롯 (머리/몸/무기/보조/장신구)
- 우측: 장착 가능 장비 목록 (그리드 인벤토리에서 필터)
- 장비 교체 시 그리드 인벤토리와 동기화

#### 3-C: 프로필 탭
- 캐릭터 배경 텍스트
- 원국 정보 (GahoSystem.getActorPillars → 4주 표시)
- 일주 천간 아이콘/이름 + 성격 요약
- 아키타입 표시 (GahoSystem.matchArchetypes)
- 신살 목록 (GahoSystem.getActorSinsal)

**산출물**: 3개 탭이 동작하는 인물 화면

### Phase 4: 프론트엔드 — 관계 (Bonds)
> 호감도 네트워크 + 원국 궁합

#### 4-A: 관계망 시각화
- 중앙에 선택 캐릭터, 주변에 다른 파티원 원형 배치
- 연결선: 호감도 수준에 따라 색상/굵기 변화
  - 높은 호감: 밝은 금색, 굵은 선
  - 보통: 회색, 가는 선
  - 낮은 호감: 붉은색
- 캐릭터 클릭 시 상세 패널 전환

#### 4-B: 원국 궁합 분석
- 두 캐릭터 선택 시 궁합 패널 표시
- GahoSystem.getElementAffinity(a, b) 기반 상성 판정
- 추가: 일주 천간 간 합/충/형/파 분석 (GahoSystem 확장 필요)
  - `getCompatibility(actorIdA, actorIdB)` API 추가
  - 천간합(干合), 지지합(支合), 충(冲), 형(刑) 판정
  - 궁합 점수 + 한줄 해석 + 게임 내 효과 설명
- 관계 이벤트 회상 목록 (해당 쌍의 과거 이벤트 기록)

**산출물**: 관계망 화면 + 궁합 상세 패널

### Phase 5: 프론트엔드 — 소지품 (Items) 그리드 인벤토리
> 핵심 난이도 최고 구간

#### 5-A: 그리드 렌더링
- PIXI.Graphics 기반 그리드 배경 (8x6 셀)
- 셀 크기: 타일사이즈 기반 (예: 40x40px)
- 배치된 아이템은 부피만큼의 셀을 차지하며 아이콘 표시
- 빈 칸 / 점유 칸 시각 구분

#### 5-B: 드래그 앤 드롭
- 마우스: 클릭 → 아이템 픽업 → 드래그 → 놓기
  - 유효 위치: 초록 하이라이트
  - 무효 위치: 빨강 하이라이트
- 키보드: 방향키 이동 → 확인 키로 픽업/놓기
- 아이템 회전 없음

#### 5-C: 핫키 슬롯
- 그리드 하단 또는 우측에 4~6개 1x1 슬롯
- 소비 아이템만 등록 가능 (itypeId === 1)
- 핫키 등록 아이템은 SRPG 전투 중 빠른 사용 가능
- 미등록 아이템은 전투 중 별도 "가방 열기"(턴 소모 큼) 필요

#### 5-D: 캐릭터 간 아이템 이동
- 두 캐릭터 그리드를 좌우 분할 표시
- 좌측 캐릭터 ↔ 우측 캐릭터 간 드래그 이동
- 파티 디스플레이에서 캐릭터 선택으로 전환

#### 5-E: RMMZ 기본 아이템 시스템 브릿지
- $gameParty.gainItem → 그리드 인벤토리에 자동 배치 시도
  - 배치 가능 공간 없으면 알림 ("인벤토리가 가득 찼습니다")
- $gameParty.hasItem → 전 파티원 그리드 검색
- 상점/이벤트 아이템 획득 시 그리드 연동

**산출물**: 완전한 그리드 인벤토리 시스템

### Phase 6: 프론트엔드 — 전술 (Tactics)
> 파티 전술 분석 + 원국 성향 + 스킬 목록

#### 6-A: 파티 전술 개요
- 캐릭터 카드 6장 가로 배치
- 각 카드: 초상화 + 이름 + 상위 3 스킬(아이콘+이름)
- 스킬 순서: 레벨/데미지/우선도 기준 정렬

#### 6-B: 원국 전투 성향 파이 차트
- 각 캐릭터 카드 옆에 소형 파이 차트
- GahoSystem.getActorElements → 오행 분포를 전투 역할로 매핑
  - 목(木) → 공격/성장
  - 화(火) → 화력/버스트
  - 토(土) → 방어/지원
  - 금(金) → 기동/정밀
  - 수(水) → 전략/디버프
- PIXI.Graphics 파이 차트 (오행 색상 EC 맵 활용)

#### 6-C: 파티원 간 전투 궁합
- Phase 4-B의 궁합 데이터를 전술 관점으로 재해석
- 쌍별 시너지 매트릭스 (6x6 격자, 셀 색상으로 표시)
- "인접 배치 시 보너스" 등 전술 힌트

#### 6-D: 스킬 전체 목록 (토글)
- 특정 버튼(예: Shift 또는 화면 내 버튼) 누르면 팝업
- 선택 캐릭터의 습득 스킬 전체 목록
- 각 스킬: 아이콘, 이름, MP소모, 사거리, 간략 설명
- 습득 경로 표시 (누구에게 배움 / 이벤트 습득)
- 아직 미습득이지만 배울 수 있는 스킬은 회색 아이콘으로 힌트

**산출물**: 전술 분석 화면 + 스킬 목록 팝업

### Phase 7: 프론트엔드 — 일지 (Journal) + 시스템 (System)
> 비교적 단순한 메뉴들

#### 7-A: 일지
- 탭: 메인 퀘스트 | 사이드 퀘스트 | 이벤트 기록
- 각 항목: 제목, 진행 상태(진행중/완료), 설명, 관련 인물
- 주요 선택지 결과 기록 포함
- 플러그인 커맨드로 퀘스트 추가/갱신/완료

#### 7-B: 시스템
- 저장 (세이브 슬롯 선택)
- 불러오기
- 옵션 (BGM/SE 볼륨, 텍스트 속도, 화면 설정)
- 타이틀로
- 기존 RMMZ Scene_Save/Load/Options 재활용 가능

**산출물**: 일지 + 시스템 화면

### Phase 8: GahoSystem.js 확장
> 궁합 판정 API 보강

#### 8-A: getCompatibility(actorIdA, actorIdB)
- 천간합(干合) 판정: 甲己, 乙庚, 丙辛, 丁壬, 戊癸
- 지지 삼합/육합/방합 판정
- 충(冲) 판정: 子午, 丑未, 寅申, 卯酉, 辰戌, 巳亥
- 형(刑) 판정
- 종합 궁합 점수(0~100) + 관계 유형 텍스트
- 전투 시너지 보너스 수치 (인접 배치 시 적용)

#### 8-B: getCombatTendency(actorId)
- 오행 분포 → 전투 역할 비율 반환
- {attack: 30, burst: 25, defense: 20, mobility: 15, strategy: 10}
- 전술 메뉴 파이 차트의 데이터 소스

**산출물**: GahoSystem API 확장

### Phase 9: SRPG 전투 연동
> 핫키 시스템 + 배치 페이즈 + 궁합 보너스

#### 9-A: 배치 페이즈 (Deployment Phase) ✅
- SRPG 전투 시작 직후, 전투 전에 삽입
- 맵 위 배치 가능 위치(이벤트/리전으로 지정) 하이라이트
- 파티원을 선택 → 배치 위치 클릭으로 배치
- 전원 배치 완료 시 "전투 개시" 확인
- SM에 `deployment` 페이즈 추가: idle → deployment → banner(전투시작) → ...

#### 9-B: 핫키 전투 사용 ✅
- SRPG 전투 중 유닛 턴에 "핫키" 서브메뉴
- 핫키 슬롯에 등록된 아이템 즉시 사용 (보조행동 소모)
- 미등록 아이템은 "가방" 메뉴 → 메인행동 소모

#### 9-C: 인접 궁합 보너스 ✅
- 전투 중 인접 아군과의 궁합 체크
- getCompatibility 결과에 따라 소규모 스탯 보너스
- UI: 유닛 정보 패널에 궁합 보너스 표시

**산출물**: 배치 페이즈 + 핫키 전투 연동 + 궁합 보너스

## 5. 비전투 능력치 축 후보 (확정 필요)

**십성 5축 (PARAM_AXES)**: 주체성, 사교성, 현실감각, 자기절제, 계획성
**오행 5축 (ELEM_AXES)**: 공감능력, 몰입능력, 정서안정, 판단력, 학습능력
