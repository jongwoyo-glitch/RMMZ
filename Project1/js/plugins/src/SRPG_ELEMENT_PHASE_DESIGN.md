## 1. 아키텍처 개요

DOS2의 내부 데이터 구조(`SurfaceType` 열거형)에서 도출한 3계층 모델을 SRPG에 적용한다.

```
┌──────────────────────────────────────────────┐
│ Layer 3: 변형자 (Modifier)                    │
│   일반(Normal) / 축복(Blessed) / 저주(Cursed)  │
│   빛·어둠 스킬로 전환                          │
├──────────────────────────────────────────────┤
│ Layer 2: 원소 오버레이 (Element Overlay)       │
│   없음 / 전기(Electrified) / 동결(Frozen)      │
│   천둥·얼음 스킬로 전환. 상호 배타적            │
│   액체 계열에만 적용 가능                       │
├──────────────────────────────────────────────┤
│ Layer 1: 기반 타입 (Base Type)                │
│   화염, 수면, 피, 독늪, 기름, 진흙, 거미줄, 용암 │
│   원소 스킬로 생성 / 대체 / 소멸                │
└──────────────────────────────────────────────┘

장판 ID = [Base] × [Overlay] × [Modifier]
예: 저주 감전수 = [수면] × [전기] × [저주]
```

## 2. RMMZ 속성 목록

| ID | 속성명 | 장판 역할 | 오버레이 역할 | 변형자 역할 |
|----|--------|-----------|---------------|-------------|
| 1 | 물리 | — | — | — |
| 2 | 불 | Base 생성(화염) + 가연물 소멸 | — | — |
| 3 | 얼음 | — | **Frozen 오버레이** (액체→동결) | — |
| 4 | 천둥 | — | **Electrified 오버레이** (액체→감전) | — |
| 5 | 물 | Base 생성(수면) + 화염 소멸 | — | — |
| 6 | 흙 | Base 생성(진흙) + 수면 변환 | — | — |
| 7 | 바람 | — | — (구름 확산/제거) | — |
| 8 | 빛 | — | — | **Blessed 변형자** |
| 9 | 어둠 | — | — | **Cursed 변형자** |
| 10 | 독 | Base 생성(독늪) + 가연성 | — | — |

각 속성의 장판 시스템 내 역할:
- **Base 생성자**: 불, 물, 흙, 독 — 자기 고유 장판을 만듦
- **Overlay 적용자**: 얼음, 천둥 — 남의 장판 위에 속성을 얹음
- **Modifier 적용자**: 빛, 어둠 — 변형자 레이어를 전환
- **촉매/제거자**: 바람 — 구름 확산·제거. 물리 — 동결/감전 해제+추가피해
- **비원소 Base**: 기름, 피, 거미줄 — 스킬/환경으로 생성

## 3. Layer 1: 기반 타입 — 지면 장판 완전 목록

### 3-A. 원소 기반 장판

| 기반 타입 | 속성 | 진입 효과 | 오버레이 허용 | 변형자 허용 |
|-----------|------|-----------|---------------|-------------|
| **화염** (Fire) | 불 | 화상 DoT | ✕ | ✅ N/B/C |
| **수면** (Water) | 물 | 젖음 | ✅ 전기·동결 | ✅ N/B/C |
| **독늪** (Poison) | 독 | 중독 DoT | ✕ | ✅ N/B/C |
| **진흙** (Mud) | 흙 | 이동↓↓, 속박 | ✅ 전기만 | ✅ N/B/C |

### 3-B. 비원소 기반 장판

| 기반 타입 | 분류 | 진입 효과 | 오버레이 허용 | 변형자 허용 |
|-----------|------|-----------|---------------|-------------|
| **기름** (Oil) | 무속성 | 둔화, 유침 | ✕ | ✅ N/B/C |
| **피** (Blood) | 무속성 | — (Leech) | ✅ 전기·동결 | ✅ N/B/C |
| **거미줄** (Web) | 무속성 | 속박(갑옷무시) | ✕ | ✅ N/B/C |
| **용암** (Lava) | 불(강화) | 화상+이동↓+DEF↓ | ✕ | ✕ |

### 3-C. 오버레이 허용 규칙

오버레이는 **액체 계열**(물기가 있어 전도/동결 가능한 것)에만 적용.

| 기반 타입 | 전기(Electrified) | 동결(Frozen) | 이유 |
|-----------|-------------------|--------------|------|
| 수면 | ✅ → 감전수 | ✅ → 빙판 | 물 = 전도체+동결 가능 |
| 피 | ✅ → 감전피 | ✅ → 동결피 | 피 = 액체, 수면과 동일 |
| 진흙 | ✅ → 감전진흙 | ✕ | 젖은 흙 = 전도체. 그러나 동결은 진흙 전체를 얼리지 못함 |
| 화염 | ✕ | ✕ | 불 = 전도/동결 불가 |
| 독늪 | ✕ | ✕ | 독 = 화학물. 전도 불가. 동결 불가(점성) |
| 기름 | ✕ | ✕ | 기름 = 부도체+비결빙 |
| 거미줄 | ✕ | ✕ | 고체. 전도/동결 무관 |
| 용암 | ✕ | ✕ | 제거 불가 |

전기와 동결은 **상호 배타적**: 감전수에 얼음 → 빙판(전기 소멸). 빙판에 천둥 → 감전수(동결 해제+녹음+전기화).

**1히트 1레이어 원칙**: 하나의 스킬은 하나의 레이어만 처리한다. 동결피에 불을 쓰면 Frozen 해제→피로 복귀(L2 처리)에서 끝. 피를 증발시키려면(L1 처리) 별도 히트가 필요. 이는 연쇄 반응의 예측 가능성을 보장한다.

## 4. Layer 2: 원소 오버레이 — 전기·동결 조합 전체

### 4-A. 전기(Electrified) 오버레이 조합

| 기반 | + 전기 오버레이 | 결과명 | 추가 효과 |
|------|-----------------|--------|-----------|
| 수면 | SurfaceWaterElectrified | **감전수** | 진입 시 감전. 전기 피해 |
| 피 | SurfaceBloodElectrified | **감전피** | 진입 시 감전. 전기 피해 |
| 진흙 | SurfaceMudElectrified | **감전진흙** | 속박 + 감전 |
| 수증기(☁) | CloudSteamElectrified | **감전증기** | 감전. 전기 피해 |
| 피구름(☁) | CloudBloodElectrified | **감전피구름** | 감전 |

### 4-B. 동결(Frozen) 오버레이 조합

| 기반 | + 동결 오버레이 | 결과명 | 추가 효과 |
|------|-----------------|--------|-----------|
| 수면 | SurfaceWaterFrozen | **빙판** | 넘어짐(KD) 확률. 이동↓ |
| 피 | SurfaceBloodFrozen | **동결피** | 넘어짐(KD) 확률 |

빙판의 정체: 독립 장판이 아니라 **수면의 동결 오버레이**. 불로 녹이면 수면으로 복귀.

### 4-C. 오버레이 전환 규칙

| 현재 상태 | + 천둥 스킬 | + 얼음 스킬 | + 불 스킬 |
|-----------|-------------|-------------|-----------|
| 수면 (None) | → 감전수 (Elec) | → 빙판 (Frozen) | → 소멸+수증기 |
| 감전수 (Elec) | ↑ 유지 | → 빙판 (Elec→Frozen) | → 소멸+수증기 |
| 빙판 (Frozen) | → 감전수 (Frozen→해제→Elec) | ↑ 유지 | → 수면 (Frozen 해제) |
| 피 (None) | → 감전피 (Elec) | → 동결피 (Frozen) | → 소멸+수증기 |
| 감전피 (Elec) | ↑ 유지 | → 동결피 (Elec→Frozen) | → 소멸+수증기 |
| 동결피 (Frozen) | → 감전피 (Frozen→해제→Elec) | ↑ 유지 | → 피 (Frozen 해제) |
| 진흙 (None) | → 감전진흙 (Elec) | — (동결 불가) | → 경화 (건조) |
| 감전진흙 (Elec) | ↑ 유지 | — | → 경화 (전기 소멸) |

## 5. Layer 3: 변형자 — 축복·저주 조합 전체

### 5-A. 축복(Blessed) 장판

아군에게 버프, 적에게는 원래 디버프 유지 (SRPG 독자 설계: 아/적 구분).

| 기반×오버레이 | 축복 변형 | 아군 효과 | 적 효과 |
|---------------|-----------|-----------|---------|
| 화염 | **성화** | HP 치유, 동결 면역 | 성속성 피해 |
| 수면 | **성수** | HP 회복, 디버프 정화 | 젖음 |
| 감전수 | **축복감전수** | Haste(행동력↑) | 감전 |
| 빙판 | **축복빙판** | 미끄러짐 면역, 마법방어↑ | 넘어짐 |
| 독늪 | **축복독늪** | Regeneration(재생) | 중독 |
| 진흙 | **축복진흙** | 이동 페널티 없음 | 속박 |
| 감전진흙 | **축복감전진흙** | Haste | 속박+감전 |
| 기름 | **축복기름** | Fortify(방어↑), 상태이상 정화 | 둔화 |
| 피 | **축복피** | HP 회복 | — |
| 감전피 | **축복감전피** | Haste | 감전 |
| 동결피 | **축복동결피** | 마법방어↑ | 넘어짐 |
| 거미줄 | **축복거미줄** | Haste(가속) | 속박 |

### 5-B. 저주(Cursed) 장판

효과 강화 + 일반 원소 반응으로 제거 불가. 빛(정화) 필요.

| 기반×오버레이 | 저주 변형 | 효과 변화 | 제거 저항 |
|---------------|-----------|-----------|-----------|
| 화염 | **업화(Necrofire)** | 화상 DoT 2배 | 물/얼음 소화 불가 |
| 수면 | **저주수** | **부패** 부여 + 수속성 피해 | 불로 증발 불가 |
| 감전수 | **저주감전수** | 감전(Tier2) 즉시 | — |
| 빙판 | **저주빙판** | 동결(Tier2) 확률, 갑옷 무시 | 불로 녹지 않음 |
| 독늪 | **저주독늪** | 중독 + 산성(물리방어↓) | — |
| 진흙 | **저주진흙** | 속박 + 상태이상 저항↓ | — |
| 감전진흙 | **저주감전진흙** | 감전(Tier2) 즉시 + 속박 | — |
| 기름 | **저주기름** | 둔화 + 이동 시 폭발 확률 | — |
| 피 | **저주피** | **부패** 부여 | — |
| 감전피 | **저주감전피** | 감전(Tier2) 즉시 | — |
| 동결피 | **저주동결피** | 동결(Tier2) 확률, 갑옷 무시 | 불로 녹지 않음 |
| 거미줄 | **저주거미줄** | 속박 100%(확정), 갑옷 무시 | — |

### 5-C. 축복·저주 구름

| 구름 기반×오버레이 | 축복 효과 | 저주 효과 |
|--------------------|-----------|-----------|
| 수증기 | HP 치유 | **부패** 부여 |
| 감전증기 | Haste | 감전(Tier2) 즉시 |
| 화염구름 | HP 치유, 동결 면역 | 업화(Necrofire) |
| 독구름 | Regeneration | 산성+중독 이중 |
| 연기 | **은신** 부여 | **실명+질식**(스킬 불가) |
| 눈구름 | 아군 한기 면역 | 동결 확률↑ |
| 먼지 | 아군 HIT 페널티 면역 | HIT↓↓ 강화 |
| 폭풍 | 아군 밀림 면역 | 밀림 강화 + HP↓ |
| 피구름 | HP 치유 | **부패** 부여 |
| 감전피구름 | Haste | 감전(Tier2) 즉시 |

### 5-D. 변형자 전환 규칙

```
일반 + 빛 스킬 → 축복
일반 + 어둠 스킬 → 저주
축복 + 어둠 스킬 → 일반 (상쇄)
저주 + 빛 스킬 → 일반 (정화)

2단계 해제:
  저주화염(업화) → 빛(정화) → 일반 화염 → 물(소화) → 소멸
  저주빙판 → 빛(정화) → 일반 빙판 → 불(녹임) → 수면
```

### 6-C. 총 집계

| 분류 | 기반수 | 오버레이 | 변형자 | 합계 |

## 7. 유닛 상태이상

### 7-A. 2단계 승급 상태 (Tier 시스템)

| Tier 1 (약) | → 승급 조건 | Tier 2 (강, CC) | 속성 |
|-------------|-------------|-----------------|------|
| **온기** | 불 추가 피격 | **화상** (DoT) | 불 |
| **한기** | 얼음 추가 피격 | **동결** (행동불가) | 얼음 |
| **대전** | 천둥 추가 피격 | **감전** (행동불가) | 천둥 |

승급 규칙:
- Tier 1 상태에서 같은 속성 피격 → Tier 2 승급

- Tier 2는 행동불가(CC). 물리 피격으로 해제 + 추가 피해

### 8-C. 장판 + 빛/어둠 스킬 (Layer 3 변형자 전환)

모든 장판(용암 제외)에 일괄 적용:

| 현재 변형자 | +빛 | +어둠 |
|-------------|-----|-------|
| Normal | → **Blessed** | → **Cursed** |
| Blessed | — (이미 축복) | → **Normal** (상쇄) |
| Cursed | → **Normal** (정화) | — (이미 저주) |

저주 장판의 핵심: **일반 원소 반응이 거부됨.** 빛으로 먼저 정화 → Normal → 그 다음 원소 반응.
예: 업화(저주화염) + 물 → ❌ 반응 거부. 업화 + 빛 → 화염(Normal). 화염 + 물 → ✅ 수증기+소멸.

**구름에도 동일 적용**: 구름도 장판과 같은 변형자 전환 규칙(Normal↔Blessed↔Cursed)을 따른다. 저주 구름도 일반 원소 반응을 거부하며, 빛으로 정화 후 원소 반응 가능.

## 10. 연쇄 반응 시나리오

### 시나리오 1: 기본 연쇄 (기름+불+독)
```
기름 장판 → 불 스킬 → ◎폭발+화염+연기(☁)
→ 인접 독늪에 화염 번짐 → ◎폭발+화염 확산+독구름(☁)
```

### 시나리오 2: 오버레이 연쇄 (물→천둥→얼음)
```
수면 생성 → 적 다수 수면 위(젖음)
→ 천둥 → 수면→감전수[L2 Elec] + 젖음 유닛 즉시 감전(T2)
→ 얼음 → 감전수→빙판[L2 Elec→Frozen] + 미끄러짐
```

### 시나리오 3: 변형자 활용 (어둠→부패→빛)
```
수면 + 어둠 → 저주수[L3 Cursed]
→ 적 진입 → 부패 부여
→ 아군 힐러가 부패 적에 치유 → 피해!
→ 빛 → 저주수→수면[L3 Cursed→Normal] → 이제 정상 반응 가능
```

### 시나리오 4: 2단계 해제
```
업화(저주화염) + 물 → ❌ 저주가 원소 반응 거부
→ 빛(정화) → 업화→화염[L3 Cursed→Normal]
→ 물 → ✅ 화염 소멸 + 수증기(☁)
```

### 시나리오 5: 언데드 역이용
```
적 언데드가 독늪 위에서 HP 회복 중
→ 빛 → 독늪→축복독늪[L3 Blessed]
→ 축복독늪 = 아군 재생, 적 중독 → 언데드에게 재생은 피해!
```

## 11. 구현 데이터 구조

```javascript
// === 장판/구름 객체 ===
SrpgSurface {
  baseType:   "fire"|"water"|"blood"|"poison"|"oil"|"mud"|"web"|"lava"
  overlay:    "none"|"electrified"|"frozen"   // Layer 2
  modifier:   "normal"|"blessed"|"cursed"     // Layer 3
  duration:   Number                          // 잔여 턴
  ownerId:    Number                          // 설치자 (축복 아/적 구분용)
}

SrpgCloud {
  baseType:   "steam"|"firecloud"|"poisoncloud"|"smoke"|"snow"|"dust"|"storm"|"bloodcloud"|"explosion"|"frostexplosion"
  overlay:    "none"|"electrified"            // Frozen 구름은 없음
  modifier:   "normal"|"blessed"|"cursed"
  duration:   Number
}

// === 조합 ID 산출 ===
// surfaceId = hash(baseType, overlay, modifier)
// 반응 테이블 참조: reactions[existingBase][incomingElement] → result

// === Layer별 반응 처리 순서 ===
function applyElementToSurface(surface, element) {
  // 1. 저주 장판이면 → 원소 반응 거부 (빛/어둠만 처리)
  if (surface.modifier === "cursed" && element !== LIGHT && element !== DARK) {
    return; // 반응 거부
  }
  
  // 2. Layer 3: 빛/어둠이면 → 변형자만 전환
  if (element === LIGHT || element === DARK) {
    switchModifier(surface, element);
    return;
  }
  
  // 3. Layer 2: 천둥/얼음이면 → 오버레이 전환 (대상이 액체일 때만)
  if (element === THUNDER || element === ICE) {
    if (isLiquidBase(surface.baseType)) {
      switchOverlay(surface, element);
      return;
    }
    // 액체가 아니면 L1으로 fallthrough (예: 화염+얼음→소멸)
  }
  
  // 4. Layer 1: 기반 타입 변환/소멸
  transformBase(surface, element);
}

// === 오버레이 전환 로직 ===
function switchOverlay(surface, element) {
  if (element === THUNDER) {
    if (surface.overlay === "frozen") {
      // Frozen → 해제(녹음) → Electrified
      surface.overlay = "electrified";
    } else {
      surface.overlay = "electrified";
    }
  }
  if (element === ICE) {
    if (surface.overlay === "electrified") {
      // Electrified → Frozen (전기 소멸)
      surface.overlay = "frozen";
    } else {
      surface.overlay = "frozen";
    }
  }
}

// === 변형자 전환 로직 ===
function switchModifier(surface, element) {
  if (element === LIGHT) {
    if (surface.modifier === "cursed") surface.modifier = "normal";   // 정화
    else if (surface.modifier === "normal") surface.modifier = "blessed";
  }
  if (element === DARK) {
    if (surface.modifier === "blessed") surface.modifier = "normal";  // 상쇄
    else if (surface.modifier === "normal") surface.modifier = "cursed";
  }
}

// === 언데드 판별 ===
function applyHealing(unit, amount) {
  if (unit.isUndead()) amount = -amount;  // 치유→피해
  if (unit.hasState(STATE_DECAYING)) amount = -Math.abs(amount); // 무조건 피해
  unit.gainHp(amount);
}

function applyPoison(unit, amount) {
  if (unit.isUndead()) {
    if (unit.hasState(STATE_DECAYING)) {
      unit.gainHp(-amount);  // 이중반전: 독도 피해
    } else {
      unit.gainHp(amount);   // 독=치유
    }
  } else {
    unit.gainHp(-amount);    // 일반: 독=피해
  }
}
```
