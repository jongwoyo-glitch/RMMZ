# 스킬 범위 에디터 재설계 + 페이즈 애니메이션 프리뷰

## 1. 현황 및 문제점

### 1-1. 현재 SRE (Skill Range Editor) 구조
- Reach 그리드 (21×21, 파란색) + Area 그리드 (7×7, 주황색) **나란히** 배치
- Reach = "어디를 조준할 수 있는가" (타겟 가능 범위)
- Area = "타겟 주변에 무엇이 맞는가" (효과 범위)
- `srpgRotate` 체크박스로 방향 회전 여부만 제어

### 1-2. 핵심 문제
1. **방사형 vs 탄착형 구분 불명확** — 편집할 때 "이 셀이 뭘 의미하는지" 직관적이지 않음
2. **방사형 스킬에서 Reach가 불필요** — 방사형은 시전자 기준으로 영향 범위 자체가 결정됨
3. **탄착형에서 Reach와 Area의 관계가 시각적으로 끊김** — 둘을 동시에 볼 수 없음
4. **페이즈 시퀀스 프리뷰 없음** — dash → hit → push 같은 연속 동작의 실제 결과를 볼 수 없음

## 2. 설계: 스킬 범위 표기 재설계

### 2-1. 스킬 유형 선택자 추가

SRE 상단에 **스킬 유형 토글** 추가:

```
┌─────────────────────────────────────────┐
│ 스킬 범위 유형:  [방사형 ◉] [탄착형 ○] │
└─────────────────────────────────────────┘
```

### 2-2. 방사형 (Radial) 모드

**핵심 원칙**: 시전자 중심으로 **스킬 영향 범위** 자체를 직접 편집.
탄착 가능 범위(Reach) 개념 불필요.

```
┌──────────────────────────────────┐
│ 「스킬 영향 범위」               │
│   (시전자 ★ 중심 = 실제 타격 영역) │
│  ┌───────────────────────┐      │
│  │  21×21 그리드          │      │
│  │  ★ = 시전자 (녹색)     │      │
│  │  ■ = 타격 영역 (빨간색) │      │
│  └───────────────────────┘      │
│  [srpgRotate ☑] → 4방향 미리보기 │
└──────────────────────────────────┘
```

- **단일 그리드**만 표시 (Area 그리드 숨김)
- 셀 색상: **빨간색 계열** (`#e55050`) — "타격"의 의미를 직관적으로 전달
- 중심 ★ = 시전자 위치 (녹색, 편집 불가)
- 데이터 저장: `<srpgReach:dx,dy|...>` — 기존 Reach 필드에 저장하되,
  런타임에서는 이 범위가 곧 "실제로 타격하는 영역"
- srpgRotate 활성 시: 방향별 회전 프리뷰 표시

**방사형 예시**: 전방 부채꼴 공격, 십자 마법, 자기 주변 폭발

### 2-3. 탄착형 (Impact) 모드

**핵심 원칙**: ① 탄착 가능 범위 → ② 탄착지 기준 영향 범위, 2단계 편집.

```
┌──────────────────────────────────────────────────┐
│ ① 「탄착 가능 범위」          ② 「탄착 영향 범위」  │
│  (어디에 발사할 수 있는가)     (탄착지 중심 스플래시)│
│  ┌───────────────────┐      ┌───────────┐        │
│  │  21×21 그리드      │      │  7×7 그리드│        │
│  │  ★ = 시전자 (녹색) │      │  ● = 탄착점│        │
│  │  ■ = 조준 가능(파랑)│      │  ■ = 타격 │        │
│  └───────────────────┘      └───────────┘        │
│                                                   │
│  ┌─────────────────────────────────┐              │
│  │ 「통합 미리보기」 (읽기 전용)     │              │
│  │  ★ = 시전자, ● = 선택된 탄착점   │              │
│  │  파랑 = 조준 가능, 빨강 = 타격   │              │
│  │  (탄착점 위에 마우스 → 해당 위치  │              │
│  │   기준 영향 범위 실시간 오버레이)  │              │
│  └─────────────────────────────────┘              │
└──────────────────────────────────────────────────┘
```

- **두 그리드 유지** + **통합 미리보기 추가**
- Reach 그리드: 파란색 (`#3388ff`) — "조준 가능 지점"
- Area 그리드: 빨간색 (`#e55050`) — "탄착 영향"
- **통합 미리보기**: 탄착 가능 범위 위에 마우스를 올리면 해당 지점 기준으로
  Area가 빨간색 오버레이로 표시 → 실제 게임에서의 스플래시를 즉시 확인 가능

### 2-4. 색상 체계 통일

| 요소 | 색상 | 용도 |
|------|------|------|
| 시전자 ★ | `#44cc88` (녹색) | 시전 위치 (편집 불가) |
| 조준 가능 범위 | `#3388ff` (파란색) | 탄착형의 Reach |
| 타격 영역 | `#e55050` (빨간색) | 실제 데미지가 가는 곳 |
| 타격 오버레이 | `rgba(229,80,80,0.4)` | 통합 미리보기의 스플래시 |
| 비활성 셀 | `#222540` (어두운 색) | 범위 밖 |
| 그리드 선 | `#2a2d4a` | 격자 |

### 2-5. 노트태그 호환성

기존 `<srpgReach>`, `<srpgArea>`, `<srpgRotate>` 태그를 그대로 사용.
새로운 태그 추가:

```
<srpgRangeType:radial>    ← 방사형 (기본값: radial)
<srpgRangeType:impact>    ← 탄착형
```

- **방사형**: `<srpgReach>` = 실제 타격 영역, `<srpgArea>` 무시
- **탄착형**: `<srpgReach>` = 조준 가능 범위, `<srpgArea>` = 탄착 스플래시
- 기존 스킬 하위호환: `srpgRangeType` 태그 없으면 → 자동 판별
  (Area가 비어있으면 radial, 있으면 impact)

## 3. 설계: 페이즈 애니메이션 풀 프리뷰

### 3-1. 아키텍처

```
┌──────────────────────────────────────────┐
│ 스킬 페이즈 프리뷰 캔버스                 │
│  ┌────────────────────────────────────┐  │
│  │  WebGL 캔버스 (Effekseer 렌더링)   │  │
│  │                                    │  │
│  │   ★ 시전자 스프라이트              │  │
│  │          → 이동 경로 표시           │  │
│  │              ● 타겟 스프라이트       │  │
│  │              ✦ Effekseer 이펙트     │  │
│  │                                    │  │
│  └────────────────────────────────────┘  │
│  [▶재생] [⏸일시정지] [⏹정지] [◀이전] [▶다음]│
│  Phase 1/3: dash → hit → push           │
│  ═══════●═════════════  00:02 / 00:05    │
│                                          │
│  애니메이션: [HitFire ▾] (ID:3)          │
│  속도: [100%] 스케일: [100%]             │
└──────────────────────────────────────────┘
```

### 3-2. 기술 스택

- **Effekseer WebGL 렌더러**: 프로젝트 내 `effekseer.min.js` + `effekseer.wasm` 재활용
- **WebGL 캔버스**: 프리뷰 전용 독립 WebGL 컨텍스트 생성
- **.efkefc 로드**: GitHubAdapter/로컬서버에서 바이너리 fetch
- **그리드 오버레이**: 2D 캔버스를 WebGL 캔버스 위에 레이어링

### 3-3. 프리뷰 렌더링 파이프라인

```
Animations.json 파싱
  → effectName → effects/{name}.efkefc 로드
  → effekseer.createContext() (독립 WebGL)
  → effekseer.loadEffect(arrayBuffer)
  → 프레임 루프:
     1. 그리드 배경 그리기 (2D canvas)
     2. 유닛 스프라이트 위치 업데이트
     3. effekseer.update() + effekseer.draw()
     4. UI 오버레이 (타임라인, 페이즈 표시)
```

### 3-4. 페이즈별 시각 표현

| 페이즈 유형 | 시각 표현 |
|-------------|-----------|
| `dash` | 시전자 스프라이트가 경로를 따라 이동 (보간 애니메이션) |
| `hit` | 타겟 위치에 Effekseer 이펙트 재생 + 영향 범위 하이라이트 |
| `proj` | 투사체 이미지가 시전자→타겟으로 이동 후 이펙트 재생 |
| `push/pull` | 타겟 스프라이트가 밀리거나 끌려오는 이동 |
| `chain` | 1차 타겟 → 2차 타겟 → ... 순차 이펙트 |
| `summon` | 지정 위치에 오브젝트 스프라이트 출현 |
| `terrain` | 장판 타일 스프라이트 깔림 |

### 3-5. 데이터 소스

| 항목 | 소스 |
|------|------|
| 애니메이션 목록 | `data/Animations.json` |
| Effekseer 이펙트 | `effects/{effectName}.efkefc` |
| 캐릭터 스프라이트 | `img/characters/` (SV전투용: `img/sv_actors/`) |
| 투사체 이미지 | `img/system/` 또는 커스텀 경로 |
| 사운드 | `audio/se/` (soundTimings 기반 재생) |

### 3-6. 프리뷰 설정 옵션

- **시전자 스프라이트**: 드롭다운 선택 (기본: 일반 전사)
- **타겟 스프라이트**: 드롭다운 선택 (기본: 더미)
- **그리드 크기**: 7×7 ~ 15×15
- **재생 속도**: 0.25x ~ 2x
- **페이즈 단위 실행**: 전체 재생 / 페이즈별 스텝

## 4. 구현 계획

### Phase 1: 스킬 범위 유형 분리 (방사형/탄착형) ✅
1. SRE에 유형 선택 토글 추가 (`radial`/`impact`)
2. 방사형 모드: 단일 그리드 + 빨간 타격 셀 표시
3. 탄착형 모드: 기존 2그리드 + 통합 미리보기 추가
4. 색상/레이블 통일 적용
5. `<srpgRangeType>` 노트태그 파서/시리얼라이저 추가

### Phase 2: 통합 미리보기 캔버스 ✅
1. 탄착형 모드에서 마우스 호버 시 스플래시 오버레이
2. 방사형 모드에서 방향 회전 미리보기 개선

### Phase 3: Effekseer 프리뷰 기반 구축 ✅
1. effekseer.min.js + wasm 로드 시스템 (공유 런타임 `window._efkRuntimeInit`)
2. 독립 WebGL 컨텍스트 생성 + 프리뷰 캔버스
3. .efkefc 파일 fetch (서버/GitHub 모드 대응)
4. 기본 이펙트 로드 + 재생 테스트
5. 애니메이션 드롭다운 — `State.database.animations` 기반 (직접 타이핑 없음)

### Phase 4: 페이즈 시퀀스 프리뷰 ✅
1. `SrpgPhasePreview` 모듈 — 11×11 그리드 (24px 셀) + 시전자/타겟 마커
2. Phase 타임라인 뱃지 스트립 — 클릭으로 점프, 현재 페이즈 하이라이트
3. 재생/일시정지/스텝/리셋 컨트롤 + 속도 셀렉터 (0.5x/1x/2x)
4. 9가지 페이즈 유형 애니메이션 핸들러:
   - `dash`: 시전자→타겟 인접 이동 (ease in-out)
   - `hit`: 타격 범위 셀 점멸 플래시 (area 노트태그 연동)
   - `proj`: 투사체 점 이동 → 착탄 시 hit 플래시
   - `push`/`pull`: 타겟 밀기/당기기 (dist 파라미터)
   - `escape`: 시전자 후퇴
   - `leap`: 포물선 도약
   - `swap`: 위치 교환
   - `chain`: 순차 타겟 점멸
5. SRE syncToNote / PhaseUI handleMessage에서 자동 갱신 훅

### Phase 5: 통합 및 검증 ✅
1. SRE ↔ PhasePreview 연쇄 갱신 연동 확인
2. 노트태그 입출력 양방향 검증 (srpgRangeType/Reach/Area/Phase)
3. Effekseer 런타임 공유 — 이중 초기화 방지 확인
4. JS 구문 검증 통과 + JSON 무결성 검증 통과
